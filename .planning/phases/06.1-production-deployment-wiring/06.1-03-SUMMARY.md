### Migration idempotency proof

Before Plan 03 integration check: `__drizzle_migrations` had 3 rows (0000, 0001, 0002). The production postgres volume already contained the `queues` table (orphaned from a prior `drizzle-kit push`), `posts.queue_id`/`queue_position`/`destroyed_at` columns, and queue constraints — but was missing the `post_media.transcode_status`/`transcode_error`/`deleted_at` columns. Drizzle's built-in `migrate()` aborted on `CREATE TABLE "queues"` because it cannot run partial migrations.

After the idempotent runner fix: migration 0003 completed with 11 of 18 statements skipped (duplicate objects warned, not errored) and 7 statements applied (the three missing `post_media` columns, their indexes, and related constraints). `__drizzle_migrations` now has 4 rows, and a fresh boot is a no-op.

## Deviations

Two deviations from the original plan scope, both scoped to restore the phase's stated goal of a bootable production stack:

1. **Dockerfile fix (`b4bfbe1`)** — Plan 06.1-02 modified the Dockerfile but did not touch the `pnpm deploy` lines. Those lines predate Phase 6.1 (introduced in commit `689cc11` during Phase 01). Under pnpm v10 they fail with `ERR_PNPM_DEPLOY_NONINJECTED_WORKSPACE`. Minimum-diff fix: add `--legacy` to both lines. Scoped under Phase 06.1-02 since it lives in the same file Plan 02 already owns.

2. **Migration runner fix (`c0cd098`)** — Not touched by any plan in this phase. Replaces drizzle's `migrate()` with a hand-rolled runner that skips duplicate-object SQLSTATE errors at the statement level rather than aborting the whole migration. Justification: production redeploy scenarios routinely encounter orphan schema state (dev `push`, interrupted migrations, version skew). Making the runner idempotent is a production-readiness requirement aligned with Phase 6.1's goal. Scoped under Phase 06.1-02 by convention even though the source file is in `packages/db`, because the root cause is a production-deploy concern.

Both fixes are pre-existing bugs — not regressions introduced by Wave 1.
