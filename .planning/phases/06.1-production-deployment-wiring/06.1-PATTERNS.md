### `nginx/nginx.conf` (reverse proxy, request-response)

**Analog:** The existing `location = /health` block at lines 36-42, which uses `proxy_pass http://api_backend/health;` â€” an explicit URI-with-path. The bug in `location /api/` is that it appends `/` (an explicit URI), which causes nginx to replace the matched prefix. The fix is to drop the URI entirely so nginx preserves the original request path.

**Current state** (line 45):

```nginx
            proxy_pass http://api_backend/;
```

**Required state**:

```nginx
            proxy_pass http://api_backend;
```

This is a single-character deletion (the trailing `/`). Per nginx docs: when `proxy_pass` has no URI, the original URI is passed through unchanged. Verified against `packages/api/src/routes/posts.ts`, `auth.ts`, etc. â€” all Express routes include the `/api/` prefix.

**CLAUDE.md requirement** (root) â€” "Production nginx: gzip + static asset cache headers". gzip is already configured (lines 14-17). Cache headers for Vite-hashed assets are currently missing; add a new location block, copying the structure of the existing `location /` block at lines 57-60:

**Analog** (`nginx/nginx.conf` lines 57-60):

```nginx
        location / {
            root /usr/share/nginx/html;
            try_files $uri $uri/ /index.html;
        }
```

**Required addition** (place BEFORE the `location /` catch-all so it matches first):

```nginx
        location /assets/ {
            root /usr/share/nginx/html;
            expires 1y;
            add_header Cache-Control "public, immutable";
            access_log off;
        }
```

Vite emits hashed filenames under `/assets/`, so `immutable` is safe.

### `Dockerfile` (build pipeline)

## Shared Patterns

### Factory + shutdown discipline
**Source:** `packages/worker/src/index.ts` lines 35-155; `packages/api/src/index.ts` lines 24-69
**Apply to:** the `index.ts` changes only

Two rules from root `CLAUDE.md` and `packages/api/CLAUDE.md` / `packages/worker/CLAUDE.md`:

1. All new construction inside `async function main()`. No module-level Queues, no env reads outside factories.
2. Shutdown uses per-resource try/catch with a logged error. Close order: workers â†’ queues â†’ DB â†’ Redis. For the API, there are no workers, so the order is queues â†’ Redis â†’ DB (already the existing pattern â€” just add `transcodeQueue.close()` in the queues block).

// Excerpt from packages/api/src/index.ts lines 60-63 (existing convention)

### Required-secret env var (no default, interpolated from .env)
**Source:** `docker-compose.yml` line 36 (`ENCRYPTION_KEY: ${ENCRYPTION_KEY}`), line 37 (`CSRF_SECRET: ${CSRF_SECRET}`)
