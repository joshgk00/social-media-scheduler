# Phase 06.1 Plan 02: Production Deployment Wiring Summary

Three tightly-coupled infrastructure fixes that make the production Docker Compose stack boot and serve both the SPA and API: add `SESSION_SECRET` to the api service env block, replace the empty `web_dist` named volume with a baked `web-production` nginx image, and stop stripping the `/api/` prefix in the nginx reverse proxy.

## Objective

Close audit gaps INT-02, FLOW-01, and FLOW-02 so `docker compose up --build` starts cleanly and `/api/*` + `/` both return non-404 responses, unblocking AUTH-01..07 in the validated milestone.

## Diff Summary

### `docker-compose.yml` (+4, -5)

- Inserted `SESSION_SECRET: ${SESSION_SECRET}` in the `api.environment` block between `CSRF_SECRET` and `NODE_ENV`. No `:-default` fallback — missing env fails loudly at `docker compose config`.
- Replaced the nginx service's `image: nginx:1.27-alpine` + volumes list with `build: { context: ., target: web-production }`. Volumes list dropped entirely (assets baked into image; nginx.conf COPYed in the Dockerfile stage).
- Removed `web_dist:` from the top-level `volumes:` block. Remaining volumes: `postgres_data`, `redis_data`, `media_data`.
- Healthcheck, port binding, and `depends_on` blocks unchanged.

### `Dockerfile` (+7, 0)

- Appended a `FROM nginx:1.27-alpine AS web-production` stage after `worker-production`:
  ```dockerfile
  FROM nginx:1.27-alpine AS web-production
  RUN apk add --no-cache wget
  COPY nginx/nginx.conf /etc/nginx/nginx.conf
  COPY --from=build /app/packages/web/dist /usr/share/nginx/html
  EXPOSE 80
  ```
- `wget` install is required by the compose healthcheck (`wget --spider`); it is not in the base alpine nginx image.
- No `USER` directive (port-80 bind requirement on Alpine — see Decisions).
- No explicit `CMD` — the base image's default (`nginx -g 'daemon off;'`) is correct.

### `nginx/nginx.conf` (+8, -1)

- Changed `proxy_pass http://api_backend/;` → `proxy_pass http://api_backend;` inside `location /api/`. Trailing slash was replacing the `/api/` prefix with `/`, so Express routes (all defined with `/api/` prefix) returned 404 through nginx.
- Left the separate `location = /health { proxy_pass http://api_backend/health; }` untouched — its trailing slash is intentional path rewrite because the Express health router mounts without the `/api/` prefix.
- Inserted a new `location /assets/` block between `location /api/` and `location /` with `expires 1y` + `Cache-Control: public, immutable` + `access_log off`. Ordered before the catch-all so the SPA `try_files` fallback does not intercept hashed asset requests.
