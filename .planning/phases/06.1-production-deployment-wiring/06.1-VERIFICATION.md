## Goal Alignment

Phase 6.1 was a targeted gap-closure phase addressing four integration and documentation defects from the v1.0 milestone audit:

- **INT-01**: `/api/media/*` router never mounted in production because `createApp()` received `undefined` for `storage` and `transcodeQueue`
- **INT-02**: `packages/api/src/index.ts` calls `requireEnv('SESSION_SECRET')` but the compose api service never forwarded the var
- **FLOW-01**: `nginx.conf` `/api/` location had a trailing slash on `proxy_pass` that stripped the `/api/` prefix before reaching Express
- **FLOW-02**: `docker-compose.yml` declared a `web_dist` named volume with no populator, so nginx served an empty document root
- **Audit coverage gap**: five Phase 4 requirement IDs were implemented but missing from SUMMARY frontmatter

All four codebase defects are now fixed and all five documentation IDs are claimed. The integration checkpoint (`docker compose up --build`) executed successfully end-to-end against Docker Desktop per the evidence captured in `06.1-03-SUMMARY.md`. The checked-in artifacts reflect the delivered capability.

- **Media uploads** — `POST /api/media/upload` arrives at Express → `createMediaRouter({ db, storage, transcodeQueue })` handler writes to storage backend and enqueues transcode job on `QUEUE_NAMES.transcode` → worker container consumes via same queue name + same Redis connection URL. All four flow links verified by code inspection; live check #7 confirms the router is reachable (403 CSRF gate proves the route is mounted, not the catch-all 404).
- **Session authentication** — operator `.env` → compose `environment` block → container env → `requireEnv('SESSION_SECRET')` at `index.ts:23` → `createApp({ sessionSecret })` → `createSessionMiddleware(redis, sessionSecret)`. Live API container reaches `listening` log without crash, confirming the full chain resolves.
- **Frontend serving** — build stage produces `/app/packages/web/dist` → COPY into nginx image at `/usr/share/nginx/html` → `location /` serves SPA with `try_files` fallback to `index.html` → `location /assets/` serves hashed assets with `immutable` cache. ## Deviations (Gap-Closure Fixes)

Two deviations from the original plan scope were applied during Plan 03's integration checkpoint. Both are documented in `06.1-03-SUMMARY.md` and justified as pre-existing bugs that blocked the phase's stated goal:

1. **Dockerfile pnpm v10 compatibility (`b4bfbe1`)** — Added `--legacy` flag to both `pnpm deploy` invocations in `Dockerfile:37,41`. Introduced during Phase 01; broke silently under pnpm v10 with `ERR_PNPM_DEPLOY_NONINJECTED_WORKSPACE`. Minimum-diff fix, scoped to the file Plan 02 already owned. Verdict: acceptable — required for success criterion #1 (stack must boot).

2. **Idempotent migration runner (`c0cd098`)** — Replaced drizzle's built-in `migrate()` with a hand-rolled runner at `packages/db/src/migrate.ts` that tolerates duplicate-object SQLSTATE at the statement level. Production postgres volumes routinely encounter orphan schema state from dev `push` operations or interrupted boots; drizzle's transactional abort-on-first-error contract makes those unrecoverable without manual DB surgery. Deviation produces a 167-line file in a package Phase 6.1 plans did not declare as in-scope. Verdict: acceptable — necessary for success criterion #1 in realistic production redeploy scenarios; scope widening is bounded and justified. See open follow-ups for related risks.
