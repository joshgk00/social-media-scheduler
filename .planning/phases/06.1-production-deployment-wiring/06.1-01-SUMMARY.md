# Phase 06.1 Plan 01: Wire Storage + Transcode Queue Into API Bootstrap — Summary

One-liner: Wired `createStorageBackend()` and `QUEUE_NAMES.transcode` BullMQ queue into `packages/api/src/index.ts` so the `/api/media/*` router mounts in production, closing audit gap INT-01.

## What Was Built

The API entry point `packages/api/src/index.ts` previously constructed `publishQueueService` and `notificationQueue` but never instantiated a storage backend or a transcode queue. As a result, `createApp()` received `undefined` for both `storage` and `transcodeQueue`, and the conditional gate at `app.ts:86` — `if (storage && transcodeQueue)` — evaluated false, so `createMediaRouter` was never mounted. Every request to `/api/media/*` returned 404 in production, breaking MEDIA-01 through MEDIA-08.

The fix replicates the working pattern from `packages/worker/src/index.ts:21,78-79`. One import was added, `storage` and `transcodeQueue` are now constructed inside `main()` using the same `defaultJobOptions` shape already used by `notificationQueue`, both are passed into `createApp()`, and the shutdown handler now closes the transcode queue before Redis using the existing one-liner try/catch style.
