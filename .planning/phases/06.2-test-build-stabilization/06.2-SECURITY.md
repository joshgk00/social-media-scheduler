# Phase 06.2 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| `.env.example` template | Dev-only credentials checked into source as documentation | Public-knowledge dev DB creds (scheduler/phase2password!) |
| `packages/db/drizzle/*.sql` → `runMigrations` | SQL files read from filesystem and passed to `reserved.unsafe()` | DDL statements generated by drizzle-kit; no user input crosses this boundary |
| Postgres session boundary | Advisory lock scope — lock acquired on `sql.reserve()`-pinned connection, released in finally on same session | Session-scoped `pg_advisory_lock` key |
| Process boundary (multi-replica boot) | Two containers starting the same migration serialize via advisory lock | Migration journal reads/writes |
| Test harness → `DATABASE_URL_TEST` | Dev-only Postgres credentials for test suite | Non-production connection string |
| `runWithStagedMigrations` → repo `packages/db/drizzle/` | Temporarily overwrites source-controlled files during test execution; restores via three-layer safety net | Migration SQL files (temporary swap) |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation | Status |
|-----------|----------|-----------|-------------|------------|--------|
| T-06.2-P01-01 | Information Disclosure | `.env.example` | accept | Dev-only compose credentials, not a secret per project-infrastructure.md | closed |
| T-06.2-P01-02 | Tampering | `package.json` pretest hook | accept | `pnpm -r build` runs tsc only — no arbitrary code beyond what `pnpm install` already trusts | closed |
| T-06.2-P01-03 | Denial of Service | `pretest` slow-build | accept | ~15s build time acceptable; targeted `pnpm --filter` runs bypass root pretest | closed |
| T-06.2-01 | Tampering | Concurrent migrators race on `__drizzle_migrations` (H-01) | mitigate | Blocking `pg_advisory_lock(MIGRATION_LOCK_KEY)` on `sql.reserve()`-pinned connection; test scenario 6 proves serialization via shared order[] array | closed |
| T-06.2-02 | Denial of Service | Partial-migration crash loop (H-02) | mitigate | Per-migration manual BEGIN/SAVEPOINT/COMMIT wraps statement loop + journal INSERT; test scenario 5 proves rollback via planted-hash absence assertion | closed |
| T-06.2-03 | Tampering | SQLSTATE mis-swallowing (M-02) | mitigate | Narrowed `DUPLICATE_OBJECT_CODES` to `42P07, 42P06, 42710, 42701`; removed `42P03, 42P04, 42P05, 42723`; test scenarios 3 & 4 prove tolerance boundaries | closed |
| T-06.2-04 | Information Disclosure | `defaultLogger` JSON output | accept | postgres-js does not echo DATABASE_URL in error shapes; `statementPreview` truncated to 120 chars | closed |
| T-06.2-05 | Malicious Code | `reserved.unsafe(statement)` executes SQL from `.sql` files | accept | Trust boundary: files generated by drizzle-kit, reviewed in PR, checked into repo; no user input path | closed |
| T-06.2-06 | Tampering | Lock key collision with future `pg_advisory_lock` call site | accept | Grep-audited 2026-04-16: no other advisory lock call sites in repo; frozen-forever comment warns future reviewers | closed |
| T-06.2-P03-01 | Tampering | `runWithStagedMigrations` temporarily overwrites real `drizzle/` files | mitigate | Three-layer restoration: (a) try/finally, (b) process.once('exit'), (c) process.once('SIGINT'); plus beforeEach/afterEach SHA-256 snapshot drift detector | closed |
| T-06.2-P03-02 | Information Disclosure | Test credentials echoed in logs | accept | Dev-only compose credentials; captureLogger records only logger output, never the URL | closed |
| T-06.2-P03-03 | Denial of Service | Real-Postgres tests require Docker running | accept | Documented as test environment precondition; `passWithNoTests: true` allows partial runs | closed |
