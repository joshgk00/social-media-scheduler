### CR-01: SQL injection via unparameterized hash and entry.when in INSERT

**Files modified:** `packages/db/src/migrate.ts`

**Applied fix:** Replaced string-interpolated INSERT with parameterized query using `client.unsafe(sql, params)` pattern. Values `hash` and `entry.when` are now passed as positional parameters `$1` and `$2` instead of being interpolated into the SQL string.

### WR-01: Module-level shared client in migrate.test.ts inhibits parallel test isolation

**Files modified:** `packages/db/src/__tests__/migrate.test.ts`
**Commit:** 23d3cb8
**Applied fix:** Added `process.once('exit')` handler that calls `testClient.end()` as a safety net for cases where the test runner hard-kills the worker or an unhandled error bypasses the `afterAll` cleanup path.

### WR-02: require('node:fs') inside an ESM module

**Files modified:** `packages/db/src/__tests__/migrate.test.ts`
**Commit:** 23d3cb8

The function now uses the module-level `writeFileSync` import directly. Committed alongside WR-01 as both changes were in the same file.

### WR-03: MigrationLogger interface has no error level

**Files modified:** `packages/db/src/migrate.ts`, `packages/db/src/__tests__/migrate.test.ts`

Updated test helpers (`captureLogger`, `makeOrderingLogger`, `CapturedLog` interface) to include the `error` level so they satisfy the updated interface.
