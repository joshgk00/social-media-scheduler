## Deviations from Plan

### Deviation 1: No failing tests found

**Found during:** Task 2

**Issue:** The plan expected 4 failing tests in `packages/api/src/__tests__/*post-lifecycle*.test.ts`.
On execution, the API test suite showed 291 tests, all passing, with no post-lifecycle test files.

**Root cause:** The research (2026-04-16) described test failures from stale dist + mock-db regression.
By the time the worktree was created at commit `4d535c3` (also 2026-04-16), either:
- The stale dist had been rebuilt, eliminating the 26 dist-related failures
- The 4 mock-db `.returning()` failures were in the worker package (not API), where the worker mock-db already had resolving `.returning()` support

**Fix:** Documented the actual state in the captured-assertions artifact (4 hypothetical assertion shapes
that WOULD fail if API post-lifecycle service tests were added). The Task 3 fix was still applied because:
- The current API mock-db genuinely lacks resolving `.returning()` on `updateChainable()`
- Future API tests exercising `db.update().set().where().returning()` via `createMockDb()` would fail without it
- The fix aligns mock-db with the worker pattern and matches the plan's D-10 decision

**Files modified:** `.planning/phases/06.2-test-build-stabilization/06.2-01-CAPTURED-ASSERTIONS.md`

**Impact:** Task 3 was still executed as planned — the mock-db extension is correct and all 291 tests
continue to pass after the change.
