**Orchestration shape to preserve** (lines 69-92):
```typescript

  const migrationsFolder = resolve(__dirname, '../drizzle');
  const client = postgres(databaseUrl, { max: 1 });
  try {
    await ensureMigrationsTable(client);
    const journal = await readJournal(migrationsFolder);
    const applied = await readAppliedHashes(client);

    for (const entry of journal.entries) {
      // ... pending-hash filter ...

  }
}
```
Signature `(databaseUrl: string, logger: MigrationLogger = defaultLogger)` **must not change** — `packages/api/src/index.ts:27` calls `runMigrations(DATABASE_URL)` and the API boot path is out of scope for 6.2.

**Per-statement duplicate-tolerance pattern to keep** (lines 132-147):
```typescript
for (const statement of statements) {
```

**Change:** swap `client.unsafe(statement)` for `tx.unsafe(statement)` inside `reserved.begin(async (tx) => { ... })`. Keep the try/catch + SQLSTATE classification identical; the `throw err` branch now aborts the transaction instead of leaving a half-applied migration.

**Error-logging pattern to preserve** (lines 53-56, 139-143, 155-159):

I-03 is deferred — **do not swap `console` for pino**. Log shape stays the same.

**DUPLICATE_OBJECT_CODES narrowing (D-06)** — current set at lines 23-32 contains 8 codes; new set must be exactly:
```typescript
const DUPLICATE_OBJECT_CODES = new Set([
  '42P07', // duplicate_table / duplicate_index
  '42P06', // duplicate_schema
```

Drop `42P03`, `42P04`, `42P05`, `42723`. Update the comment block above the set to match.

**Cleanup discipline** (lines 89-91 + `packages/db/CLAUDE.md` §Connections):
- Nested `finally` blocks unwind outside-in: `client.end()` (outer) → `reserved.release()` (middle) → `pg_advisory_unlock` (inner). Follow Research Pitfall 2 so unlock fires before release, release fires before end.
- Apply `CLAUDE.md` §Error Handling: "Resource cleanup in `finally` blocks, not sequential `await`" — each layer wraps the next in try/finally.

**Six required scenarios (D-08):**
1. Fresh DB — applies every pending migration; journal has N rows (currently 4).
2. Idempotent re-run — running `runMigrations` twice leaves the journal row count unchanged.
3. Orphan-schema baseline — pre-seed tables from migration 0000, then run; pre-existing tables tolerated via the narrowed SQLSTATE set; journal gets all 4 rows.
4. Duplicate-object tolerance inside a transaction — plant a duplicate-first-statement migration; assert transaction commits and journal row is written.
5. Real-error rollback (D-09) — plant a migration whose second statement has a broken reference; assert (a) `await expect(...).rejects.toThrow()`, (b) journal has 0 rows for that hash, (c) partial DDL from statement 1 is NOT present (`pg_catalog.pg_tables` query).
6. Concurrent-caller serialization — `Promise.all([first, second])` of two `runMigrations()` calls against the same DB; assert journal has exactly 4 rows (not 8) and the shared `order: string[]` log array shows `first:applied:0003` before `second:start`. Pattern per RESEARCH.md §Pattern 2.

**Assertion style for transaction-rollback scenarios** (new — no direct analog, but mirrors worker integration style):
```typescript
// Verify journal has zero rows for that hash.
const rows = await testClient`SELECT 1 FROM "drizzle"."__drizzle_migrations"`;
expect(rows.length).toBe(0);

// Verify no tables were created from the valid statements that came before the broken one.
const tables = await testClient`
  SELECT tablename FROM pg_catalog.pg_tables
  WHERE schemaname = 'public' AND tablename != '__drizzle_migrations'`;
expect(tables.length).toBe(0);
```

**Concurrent-caller assertion pattern (RESEARCH.md §Pattern 2):**
```typescript
const order: string[] = [];
const first = runMigrations(TEST_URL, makeLogger('first', order));
await new Promise((r) => setTimeout(r, 20));
const second = runMigrations(TEST_URL, makeLogger('second', order));
await Promise.all([first, second]);

const firstAppliedIdx = order.findIndex((e) => e.startsWith('first:applied:0003'));
const secondStartIdx = order.findIndex((e) => e === 'second:start');
expect(firstAppliedIdx).toBeLessThan(secondStartIdx);
```
Do NOT use `Date.now()` diffs for the lock ordering assertion — per RESEARCH.md Anti-Patterns §5, timing is CI-flaky. Use the shared `order` array.

**Test isolation discipline:**
- `beforeEach` → `await resetSchemas(testClient)`. Drops `drizzle` + `public` schemas, recreates `public`.
- Each planted-failure test generates a temp migrations folder (`fs/promises` + `mkdtemp`) and calls an internal `runMigrationsFromFolder` (either a new exported helper or by passing a folder arg via the `databaseUrl`+`migrationsFolder` split — planner decides).

**Testing rule to honor** (`CLAUDE.md` §Testing): "Security-critical code (middleware, auth, encryption): 100% branch coverage. Test both success AND failure paths." Migration runner qualifies. No conditional assertions — assert precondition first.

**Current API file** (`packages/api/src/__tests__/helpers/mock-db.ts` lines 1-24) — the simpler `chainable()` returns the chain as a thenable and wires `.returning()` as `vi.fn().mockReturnValue(chain)`, which never resolves to a value:

```typescript
function chainable(terminal: unknown = []) {
  const chain: Record<string, any> = {};
  const methods = ['from', 'where', 'values', 'returning', 'set', 'limit'];
  for (const method of methods) {
    chain[method] = vi.fn().mockReturnValue(chain);
  }
  chain.then = (resolve: (val: unknown) => void) => resolve(terminal);
  return chain;

export function createMockDb() {
```

**Target diff (D-10 — RESEARCH.md §Pattern 3):** track the `set()` payload, resolve `.returning()` to `[{ ...matchedRow, ...setPayload }]`. Minimal fix — extend `chainable()` OR branch `update` onto a dedicated `updateChainable()` builder:
```typescript
function updateChainable() {
  const chain: Record<string, any> = {};
  let setPayload: Record<string, unknown> = {};
  const matchedRow: Record<string, unknown> = { id: 'mock-updated-id' };

  chain.set = vi.fn().mockImplementation((patch: Record<string, unknown>) => {
    setPayload = patch;
    return chain;
  });
  chain.where = vi.fn().mockReturnValue(chain);
  chain.returning = vi.fn().mockImplementation(() =>
    Promise.resolve([{ ...matchedRow, ...setPayload }]),
  );
  chain.then = (resolve: (val: unknown) => void) => resolve(undefined);
  return chain;
}
```
Wire into `createMockDb` as `update: vi.fn().mockReturnValue(updateChainable())`.

If the tests assert specific matched-row fields, add an optional `matchedRow` injector (`updateChainable({ id: 'post-1' })`).

**Discretion (CONTEXT.md):** whether to extend `.returning()` to `insertChain` / `deleteChain` proactively. RESEARCH.md Assumption A3 recommends minimal-fix — add to updateChain only; extend on first `insertChain`/`deleteChain` failure.

## Shared Patterns

### Pattern S1 — Nested try/finally for resource cleanup
**Source:** `packages/db/src/migrate.ts` lines 72-92 (existing `try { ... } finally { await client.end(); }`) + `packages/db/CLAUDE.md` §Connections ("Cleanup in `finally` blocks — if `migrate()` throws, client still calls `.end()`").
**Apply to:** `migrate.ts` rewrite (advisory-lock release + reserved.release + client.end must nest); `helpers/test-db.ts` stop() fn.
```typescript
try {
  const reserved = await client.reserve();
  try {
    await reserved.unsafe(`SELECT pg_advisory_lock(${LOCK_KEY}::bigint)`);
    try {
      // ... migration loop ...
    } finally {
      await reserved.unsafe(`SELECT pg_advisory_unlock(${LOCK_KEY}::bigint)`);
    }
  } finally {
    await reserved.release();
  }
} finally {
  await client.end();
}
```
Outside-in unwind order: `client.end()` last, `pg_advisory_unlock` first. Per RESEARCH.md Pitfall 2.

### Pattern S2 — JSON-structured logger shape
**Source:** `packages/db/src/migrate.ts` lines 48-56 (`MigrationLogger` interface + `defaultLogger`).
**Apply to:** test-file `makeLogger(prefix, order)` helpers — must satisfy the same `info(msg, ctx)` / `warn(msg, ctx)` signature so tests can inject a logger without patching the production default.
```typescript
function makeLogger(prefix: string, order: string[]): MigrationLogger {
  return {
    info: (msg, ctx) => {
      if (msg === 'Migration applied') order.push(`${prefix}:applied:${(ctx as any).migration}`);
      // ...
    },
    warn: (msg, ctx) => { /* same shape */ },
  };
}
```
Do NOT expand `MigrationLogger` to add methods (`error`, `debug`) — I-03 (pino swap) is deferred.
