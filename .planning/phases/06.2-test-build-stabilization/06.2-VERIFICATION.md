## SC4 Advisory Lock Type — Note

The phase goal and SC4 success criterion mention `pg_try_advisory_lock` in parentheses, but the plan (06.2-02-PLAN.md) and implementation correctly use `pg_advisory_lock` (blocking acquire). `pg_try_advisory_lock` is non-blocking and would return false immediately rather than serializing callers — it cannot satisfy the "second concurrent caller waits" requirement. The implementation is correct. The Scenario 6 test proves the concurrent caller actually serializes, not merely exits with a failure.
