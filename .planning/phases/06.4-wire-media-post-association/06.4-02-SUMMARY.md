## Accomplishments

## Decisions Made

- Used `expect.objectContaining({ mediaIds: [...] })` for positive assertions rather than full object equality — the third/fourth service argument contains many other fields (`profileId`, `text`, `postVersion`, etc.), so `objectContaining` is the right matcher.
- For the "undefined mediaIds" test, destructured `mockCreatePost.mock.calls[0]` and asserted `.mediaIds === undefined` rather than using `not.toHaveProperty` — explicit is clearer.
