## Fixed Issues

### WR-02: `softDeleteMediaForPost` always returns 0 — missing `.returning()` on the Drizzle update chain

**Files modified:** `packages/api/src/services/media.service.ts`
**Commit:** 28318f1
**Applied fix:** Added `.returning({ id: postMedia.id })` to the Drizzle update chain and removed the now-unnecessary `Array.isArray` fallback. `updatedRows` is now the returned array and `.length` gives the correct count.

---

### WR-04: `associateMediaToPost` opens a redundant inner transaction when called with a `tx` handle

**Files modified:** `packages/api/src/services/media.service.ts`
**Commit:** 410da23
**Applied fix:** Removed the `db.transaction(async (tx) => { ... })` wrapper. The updates now run directly against the `db` parameter (which is already a transaction handle at both call sites in `post.service.ts`). The loop body is unchanged.

---

### WR-01: `getPostById` does not fetch media — `createPost` and `updatePost` return responses without a `media` field

**Files modified:** `packages/api/src/services/post.service.ts`
**Commit:** e493481
**Applied fix:** Added a `postMedia` select query inside `getPostById` that mirrors the existing `postTags` join — filtered to non-deleted rows (`isNull(postMedia.deletedAt)`) and ordered by `sortOrder`. The return value now includes `media: mediaRows` alongside `tags`. Also added `isNull` to the `drizzle-orm` import, which was required by the new query but missing from the import line.

---

### WR-03: `deletePost` runs soft-delete and hard-delete without a wrapping transaction — partial failure leaves media orphaned

**Files modified:** `packages/api/src/services/post.service.ts`
**Commit:** 6f1d105
**Applied fix:** Wrapped the entire body of `deletePost` in `db.transaction(async (tx) => { ... })`. Both `softDeleteMediaForPost` and `tx.delete(posts)` now use the transaction handle `tx`, as does the fallback `existingPost` lookup. If the hard-delete fails or the post is in a non-deletable state, throwing inside the transaction rolls back the soft-delete automatically. Added a brief inline comment explaining the atomicity guarantee.
