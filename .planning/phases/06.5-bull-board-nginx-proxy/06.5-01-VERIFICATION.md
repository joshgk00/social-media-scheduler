### Gaps Summary

**One gap blocking goal achievement — requires developer decision:**

**ROADMAP SC #2 vs PLAN architecture conflict.**

ROADMAP.md Phase 6.5 Success Criterion 2 states:

> "docker-compose.yml nginx service mounts the `media_data` volume so express.static can serve files"

The docker-compose.yml nginx service block has no `volumes:` entry. The `media_data` volume is mounted only on `api` (line 32) and `worker` (line 56).

However, the PLAN explicitly forbids adding the volume to nginx:

> "Do NOT add the `media_data` volume to the nginx service in docker-compose.yml (Express already has it; nginx is proxying, not serving directly)"

The PLAN's proxy architecture is correct: nginx proxies `/media/` requests to Express, which has the `media_data` volume. Express serves the files. nginx never touches the filesystem. The architecture is sound and the wiring works.

The ROADMAP SC was written assuming nginx would serve media directly (bypassing Express). That approach would require the volume on nginx and a direct `root`/`alias` directive — but it would also bypass Express path validation and auth context.

**Options:**

1. **Update ROADMAP.md** — Change SC #2 to reflect the proxy architecture: "nginx /media/ requests proxy to Express, which serves files from the media_data volume." No code change needed. Preferred if the proxy approach is the intended design.

2. **Add media_data volume to nginx service** — Add `volumes: - media_data:/app/data/media` to the nginx service and replace the proxy block with a direct `root`/`alias` directive. This matches the ROADMAP SC but bypasses Express path validation and removes the consistent proxy pattern.

3. **Add override to VERIFICATION.md** — If the proxy approach is the accepted design, add:
   ```yaml
   overrides:
     - must_have: "docker-compose.yml nginx service mounts the media_data volume"
       reason: "Proxy architecture chosen instead — nginx forwards /media/ to Express, which has the volume. Nginx serving files directly would bypass Express path validation."
       accepted_by: "joshgk00"
   ```
