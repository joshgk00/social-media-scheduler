<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> nginx | External HTTP requests enter the stack through nginx on the published port |
| nginx -> Express API | nginx forwards requests to Express; Express runs auth middleware, input validation, and file serving |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06.5-01 | Tampering | nginx /media/ location | mitigate | Path traversal attempts (e.g., `/media/../../etc/passwd`) are mitigated by Express `express.static()` which resolves paths relative to mediaDir and rejects traversal. Additionally, `LocalStorage.resolveAndGuard()` validates paths. nginx proxying preserves this protection. No nginx-level mitigation needed. |
| T-06.5-02 | Elevation of Privilege | nginx /admin/ location | mitigate | Unauthenticated access to Bull-Board is blocked by Express `requireAuth` middleware (admin.ts:43), which returns 401 without valid session. nginx proxy preserves this -- no nginx-level auth bypass is introduced. Do NOT add nginx-level auth_basic (would create double auth and potential bypass). |
| T-06.5-03 | Information Disclosure | nginx /media/ location | accept | Media file URLs contain profile IDs in the path (`/media/{profile_id}/...`). This is existing behavior from the storage path pattern (MEDIA-06). The profile ID is not sensitive PII for a single-user app. nginx access logs will include these paths (existing tech debt L-02). |
| T-06.5-04 | Spoofing | nginx /admin/ proxy headers | mitigate | X-Forwarded-For and X-Real-IP headers set by nginx ensure Express sees the real client IP for rate limiting and logging. These match the existing /api/ block pattern. |
| T-06.5-05 | Tampering | CSRF on Bull-Board POST actions | accept | Bull-Board's internal POST actions (retry job, clean queue) do not carry CSRF tokens. The existing CSRF exemption is documented (T-04-04-07 from Phase 4). Bull-Board is mounted before CSRF middleware in app.ts. This is acceptable for a single-user operator tool that requires session auth. |
</threat_model>
