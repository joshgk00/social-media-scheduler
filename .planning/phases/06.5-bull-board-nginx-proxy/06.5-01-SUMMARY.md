# Phase 06.5 Plan 01: Nginx Proxy Blocks for Media and Admin Summary

Nginx location blocks added for /media/ and /admin/ paths in both prod and dev configs, preventing the SPA catch-all from swallowing requests meant for Express static file serving and Bull-Board dashboard.

## What Changed

### nginx/nginx.conf (production)

Two new location blocks inserted between `/api/` and `/assets/`:

- `location /media/` -- proxies to `api_backend` with standard headers, no WebSocket upgrade. Matches `app.use('/media', express.static(mediaDir))` in Express.
- `location /admin/` -- proxies to `api_backend` with standard headers plus WebSocket upgrade. Matches `router.use('/admin/queues', requireAuth, serverAdapter.getRouter())` in Express.

Final location block order: /health, /api/, /media/, /admin/, /assets/, / (catch-all).

### nginx/nginx.dev.conf (development)

Same two blocks inserted between `/api/` and the catch-all `/` proxy to Vite dev server.

The existing `/api/` block uses `proxy_pass http://api_backend/;` (WITH trailing slash) to strip the `/api/` prefix in dev mode. The new `/media/` and `/admin/` blocks correctly use `proxy_pass http://api_backend;` (NO trailing slash) because Express expects the full path prefix.

Final location block order: /health, /api/, /media/, /admin/, / (catch-all to Vite).

## Decisions Made

1. **No WebSocket headers on /media/ block** -- Static file serving never uses WebSocket or SSE. Including the headers would be harmless but unnecessary noise.
2. **WebSocket headers on /admin/ block** -- Bull-Board may use WebSocket or SSE for real-time job status updates. Including upgrade headers prevents breakage if used.
3. **No trailing slash on proxy_pass for new blocks** -- Express mounts are at `/media` and `/admin/queues`. A trailing slash would strip the path prefix, breaking the route match.
