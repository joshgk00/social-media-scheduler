## Pattern Assignments

### `nginx/nginx.conf` (config, request-response)

**Analog:** Self -- the existing `/api/` location block at lines 44-55

Two new location blocks must be inserted between the `/api/` block (line 55) and the `/assets/` block (line 57). Both follow the same proxy pattern as `/api/` with minor variations.

**Proxy block pattern -- standard headers** (lines 44-55):
```nginx
        location /api/ {
            proxy_pass http://api_backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_set_header X-Request-ID $request_id;

            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
        }
```

**New `/media/` block -- copy headers, omit WebSocket upgrade:**

The `/media/` block needs the standard proxy headers and `X-Request-ID` but does NOT need WebSocket/Upgrade headers (static file serving has no WebSocket component). Copy lines 45-51 from the `/api/` block. Omit lines 52-54 (proxy_http_version, Upgrade, Connection).

Critical: `proxy_pass http://api_backend;` with NO trailing slash. Express mounts at `app.use('/media', express.static(mediaDir))` (app.ts line 91) and expects the full `/media/...` path.

**New `/admin/` block -- copy headers, include WebSocket upgrade:**

The `/admin/` block needs ALL proxy directives from the `/api/` block (lines 45-54), including WebSocket upgrade headers. Bull-Board may use WebSocket or SSE for real-time job updates.

Critical: `proxy_pass http://api_backend;` with NO trailing slash. Bull-Board base path is `/admin/queues` (admin.ts line 32) and Express expects the full path.

**Insertion point:** After line 55 (closing `}` of `/api/` block), before line 57 (opening of `/assets/` block).

**SPA catch-all that causes the current bug** (lines 64-67):
```nginx
        location / {
            root /usr/share/nginx/html;
            try_files $uri $uri/ /index.html;
        }
```
This catch-all currently swallows `/media/` and `/admin/` requests because no more-specific prefix match exists. The new location blocks fix this -- nginx longest-prefix matching selects `/media/` and `/admin/` over `/`.

### `nginx/nginx.dev.conf` (config, request-response)

**Analog:** Self -- the existing `/api/` location block at lines 44-55

Same two location blocks must be inserted, but note one critical difference in the dev config:

**Dev `/api/` block uses trailing slash** (line 45):
```nginx
        location /api/ {
            proxy_pass http://api_backend/;
```
The trailing slash strips the `/api/` prefix before forwarding to Express. This is correct for the `/api/` path in dev because Express routes don't include the `/api/` prefix.

**The new `/media/` and `/admin/` blocks must NOT use trailing slash in dev**, even though `/api/` does. Express expects the full path for both:
- `app.use('/media', express.static(mediaDir))` -- needs `/media/...` path
- `router.use('/admin/queues', requireAuth, ...)` -- needs `/admin/queues` path

**Insertion point:** After line 55 (closing `}` of `/api/` block), before line 57 (opening of `/` catch-all block).

**Dev catch-all differs from prod** (lines 57-67):
```nginx
        location / {
            proxy_pass http://web:5173;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;

            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
```

Dev proxies to Vite dev server instead of serving static files. The new blocks must still appear before this catch-all.
