**Dockerfile (INFRA-10)** -- multi-stage build:

```dockerfile
FROM node:22-alpine AS base
ENV PNPM_HOME="/pnpm"
ENV PATH="$PNPM_HOME:$PATH"
RUN corepack enable

# Development target (used by docker-compose.dev.yml)
FROM base AS development
RUN apk add --no-cache python3 make g++ linux-headers
WORKDIR /app
COPY pnpm-lock.yaml pnpm-workspace.yaml package.json ./
COPY packages/ packages/
RUN --mount=type=cache,id=pnpm,target=/pnpm/store pnpm install
EXPOSE 3000 9229

# Install build tools for native addons (argon2, sharp)
FROM base AS build-deps
RUN apk add --no-cache python3 make g++ linux-headers

# Install all dependencies
FROM build-deps AS install
WORKDIR /app
COPY pnpm-lock.yaml pnpm-workspace.yaml package.json ./
COPY packages/shared/package.json packages/shared/
COPY packages/db/package.json packages/db/
COPY packages/api/package.json packages/api/
COPY packages/worker/package.json packages/worker/
COPY packages/web/package.json packages/web/
RUN --mount=type=cache,id=pnpm,target=/pnpm/store pnpm install --frozen-lockfile

# Build all packages
FROM install AS build
COPY . .
RUN pnpm -r build

# Deploy API (production deps only)
FROM build AS api-deploy
RUN pnpm deploy --filter=@sms/api --prod /prod/api

# Deploy Worker (production deps only)
FROM build AS worker-deploy
RUN pnpm deploy --filter=@sms/worker --prod /prod/worker

# API production image
FROM base AS api-production
RUN apk add --no-cache ffmpeg wget
COPY --from=api-deploy /prod/api /app
WORKDIR /app
EXPOSE 3000
CMD ["node", "dist/index.js"]

# Worker production image
FROM base AS worker-production
RUN apk add --no-cache ffmpeg
COPY --from=worker-deploy /prod/worker /app
WORKDIR /app
CMD ["node", "dist/index.js"]
```

**nginx config (D-01, D-02, D-03, INFRA-03):**

Create `nginx/nginx.conf` -- plain HTTP reverse proxy. No TLS at the nginx layer. Cloudflare Tunnel handles TLS externally.

```nginx
worker_processes auto;

events {
    worker_connections 1024;
}

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    sendfile on;
    keepalive_timeout 65;

    # Logging
    log_format main '$remote_addr - $remote_user [$time_local] '
                    '"$request" $status $body_bytes_sent '
                    '"$http_referer" "$http_user_agent" '
                    'req_id=$request_id';
    access_log /dev/stdout main;
    error_log /dev/stderr warn;

    upstream api_backend {
        server api:3000;
    }

    server {
        listen 80;
        server_name _;

        # Health check (direct to API, no /api prefix)
        location = /health {
            proxy_pass http://api_backend/health;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }

        # API routes -- strip /api prefix
        location /api/ {
            proxy_pass http://api_backend/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_set_header X-Request-ID $request_id;

            # WebSocket support (for potential future use)
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
        }

        # Frontend (all other routes)
        location / {
            proxy_pass http://web:5173;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;

            # WebSocket support (Vite HMR in dev)
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
        }
    }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External network -> nginx | Cloudflare Tunnel terminates TLS; nginx receives plain HTTP |
| nginx -> api/web | Internal Docker network, trusted |
| .env file -> containers | Secrets loaded from environment variables |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-02 | Information Disclosure | .env / docker-compose.yml | mitigate | `.env` is gitignored. `docker-compose.yml` references vars via `${VAR}` syntax, never hardcodes secrets. `.env.example` has empty values for secrets. |
| T-1-03 | Spoofing | nginx reverse proxy | accept | Internal Docker network only. Cloudflare Tunnel provides external authentication boundary. Single-user app on controlled infrastructure. |
| T-1-04 | Denial of Service | Redis noeviction | mitigate | `maxmemory-policy noeviction` per D-10 prevents BullMQ data loss. Redis will reject writes when full rather than evicting job data. Monitor via /health endpoint. |
| T-1-05 | Tampering | Dockerfile native addons | mitigate | Build tools (python3, make, g++) installed only in build stage, not in production image. Production image is minimal Alpine with only runtime deps. |
</threat_model>
