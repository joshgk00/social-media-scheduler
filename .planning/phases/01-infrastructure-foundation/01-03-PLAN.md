    RED phase -- write these tests FIRST, all must fail:

    1. **Round-trip test**: `encrypt("my-secret-token", validKey)` produces an `EncryptedPayload` with non-empty `ciphertext`, `iv`, `authTag`, and `version === 1`. Then `decrypt(payload.ciphertext, payload.iv, payload.authTag, validKey)` returns `"my-secret-token"`.

    2. **Version parameter test**: `encrypt("token", validKey, 3)` returns payload with `version === 3`. Version is a passthrough (does not affect crypto, just stored for key rotation tracking per SEC-03).

    3. **Wrong key test**: `encrypt("token", keyA)` then `decrypt(payload.ciphertext, payload.iv, payload.authTag, keyB)` throws an error (GCM authentication failure).

    4. **Unique IV test**: Call `encrypt("same-text", key)` twice. The two returned payloads must have different `iv` values and different `ciphertext` values.

    5. **Key validation test**: `validateEncryptionKey("too-short")` throws. `validateEncryptionKey(validHexKey)` returns a 32-byte Buffer. Key must be exactly 64 hex characters (32 bytes).

    6. **Empty plaintext test**: `encrypt("", key)` and `decrypt` round-trip works (edge case -- empty string should encrypt/decrypt correctly).

    7. **Unicode plaintext test**: `encrypt("Hello World", key)` round-trips correctly (multi-byte characters).

    8. **No caching architecture test**: Verify `encrypt` and `decrypt` are pure functions -- they do NOT cache, memoize, or store any state. This is verified by checking that the module exports only functions (no class instances, no module-level Map/WeakMap, no Redis references). This satisfies SEC-04 by design.

    Use `crypto.randomBytes(32)` to generate `validKey` as a Buffer for tests. Use `crypto.randomBytes(32).toString('hex')` for hex key validation tests.

    - `packages/shared/src/encryption.ts` using Node.js built-in `crypto` module:
      - `const ALGORITHM = 'aes-256-gcm'`
      - `const IV_LENGTH = 12` (96-bit nonce, NIST recommendation for GCM)
      - `const AUTH_TAG_LENGTH = 16` (128-bit auth tag)
      - `interface EncryptedPayload { ciphertext: string; iv: string; authTag: string; version: number; }` -- all strings are hex-encoded
      - `function encrypt(plaintext: string, key: Buffer, version: number = 1): EncryptedPayload` -- generates random IV via `randomBytes(IV_LENGTH)`, creates cipher via `createCipheriv(ALGORITHM, key, iv, { authTagLength: AUTH_TAG_LENGTH })`, returns hex-encoded ciphertext + iv + authTag + version
      - `function decrypt(ciphertext: string, iv: string, authTag: string, key: Buffer): string` -- creates decipher via `createDecipheriv(ALGORITHM, key, Buffer.from(iv, 'hex'), { authTagLength: AUTH_TAG_LENGTH })`, sets auth tag via `decipher.setAuthTag(Buffer.from(authTag, 'hex'))`, returns UTF-8 string
      - `function validateEncryptionKey(hexKey: string): Buffer` -- validates hex string is exactly 64 characters, returns `Buffer.from(hexKey, 'hex')`. Throws `Error('ENCRYPTION_KEY must be exactly 64 hex characters (32 bytes)')` if invalid.
