# Phase 8 — UI Design Contract

> Visual and interaction contract for LinkedIn & Facebook Post Creation. Generated by gsd-ui-researcher. Pre-populated from `08-CONTEXT.md` (D-01 through D-15), `06-UI-SPEC.md` baseline, `packages/web/src/index.css`, `packages/web/components.json`, and the established Phase 3-7 component inventory. Phase 8 extends the existing `NewPostPage` and `EditPostPage` with platform-aware branching (D-01), adds LinkedIn and Facebook post previews matching the Twitter `TweetPreview` pattern, introduces a new `/dashboard` route hosting the rate-limit summary card (D-13), and slots a compact rate-limit usage chip into each `ProfileCard` below the existing `TokenHealthBadge`.

### Cross-Platform Profile Switch Flow (D-04)

1. User changes profile in picker.
2. New `platform` is computed from selected profile.
3. If `platform` differs from current form `platform`:
   - Compute the field-drop diff against the new platform's discriminated-union variant (D-02).
   - Truncate `text` to new char limit if needed.
   - Drop incompatible fields silently from form state.
   - Show `toast.info(...)` with the exact copy from the toast table above. Toast auto-dismisses after 6s. Single toast — not stacked.
   - Re-run preview rendering with new variant.
4. If new profile's rate limit at 100%, also show `RateLimitBanner` (block) — independent of toast.

### Rate-Limit Pre-flight on Submit (D-05, D-07)

Same pattern as Phase 4 Twitter pre-flight, extended to LI / FB:

| Condition | UI behavior |
|-----------|-------------|
| Pre-flight passes | Submit proceeds, post created. |
| Pre-flight returns warn (>= 90%) | Submit proceeds. Inline `RateLimitBanner` (warn) shown for next render. No interruption. |
| Pre-flight returns block (100%) | Submit aborts client-side before HTTP call. `RateLimitBlockError` shown as a sonner `toast.error(...)` with the platform-specific block copy. Submit button transitions to disabled. |

Worker-side pre-flight (D-05, D-07) is invisible to UI — post stays `scheduled` and auto-publishes on next scanner pass when window rolls over. UI shows no progress indicator for this; user sees status flip from `scheduled` to `published` on the existing posts list.

## Accessibility Contract

All Phase 8 UI inherits the accessibility baseline from `packages/web/CLAUDE.md`: WCAG AA contrast (4.5:1 text, 3:1 large), `<main>` landmark on every page, semantic HTML before divs, labeled inputs, visible focus rings via `--color-ring`.

| Requirement | Detail |
|-------------|--------|
| Profile picker | shadcn `Select` provides `aria-haspopup="listbox"` and keyboard nav. Selected option has descriptive label including platform: `aria-label="{accountName} on {Platform}"`. |
| Visibility selector | shadcn `RadioGroup` (Radix) provides correct ARIA roles, keyboard nav (arrow keys), and group label. The group has `aria-labelledby` pointing to the "Visibility" heading. |
| Cross-platform switch toast | sonner toasts have `role="status"` (default) — announced politely without interrupting input. |
| Rate-limit chip on ProfileCard | Composite chip has `aria-label="{Platform}: {used} of {limit} used, resets in {relative}"`. The color dot is decorative (`aria-hidden="true"`). |
| Dashboard rate-limit table | Standard `<table>` with `<thead>` and `<tbody>`. Each row's bar has `role="progressbar"`, `aria-valuenow`, `aria-valuemax`, and `aria-label="{Platform} rate limit usage"`. |
| Preview card | Wrapped in `<aside aria-label="{Platform} post preview">`. Marked as supplementary content. |
| LinkedIn visibility glyph (🌐 / 👥) | Marked `aria-hidden="true"`; the visibility label text adjacent provides accessible meaning. |
| FB image grid `+N` overlay | Has `aria-label="{N} more images not shown in preview"`. |
| URL in preview | Plain text `<span>` (not `<a>`) since clicks are intentionally disabled — no broken link target. |
| Submit disabled reasons | Disabled submit has `aria-describedby` pointing to a visually hidden element with the tooltip text. |
| Sidebar nav new entry | Existing `Sidebar.tsx` pattern: `<nav aria-label="Primary">` already in place; new link inherits accessible name from visible text. |

Color contrast verified by reusing existing `--color-success`, `--color-warning`, `--color-destructive` tokens — all already pass AA in dark mode per Phase 4 contrast audit.
