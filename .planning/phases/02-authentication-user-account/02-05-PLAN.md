Adjust the DATABASE_URL based on what's in the project's `.env` or `docker-compose.dev.yml`. Check the docker-compose file for the actual postgres service credentials and port.

**B. Build `packages/web/src/pages/login/LoginPage.tsx`:**

Per UI-SPEC `/login` contract. Replace the stub.

Layout: Same centered card as setup (max-width 400px).

Two-step flow managed by local React state (`step: 'credentials' | 'totp'`):

**Step 1 (Credentials):**
```
<Card>
  <CardHeader>
    <CardTitle> "Social Media Scheduler" (Display)
  </CardHeader>
  <CardContent>
    <form> (React Hook Form with zodResolver(loginSchema))
      1. Email input (type="email", label="Email")
      2. Password input (type="password", label="Password")
      3. Alert for errors (inline, below form):
         - Invalid credentials: "Invalid email or password."
         - Rate limit: "Too many failed attempts. Try again in {N} minutes."
      4. "Sign In" primary button (full width)
         - Disabled while fields empty, shows spinner during submit
      5. "Forgot password?" link below button -> /recover
    </form>
  </CardContent>
</Card>
```

On successful login:
- If response `requiresTwoFactor === true`: transition to Step 2
- If `requiresTwoFactor === false`: redirect to `redirectAfterLogin` from Zustand store (or `/` if null), clear the redirect

On error:
- 401: show "Invalid email or password." alert
- 429: show "Too many failed attempts. Try again in 15 minutes." alert (parse rate limit headers if available)

**Step 2 (TOTP, conditional):**
```
<Card> (same card, content changes)
  <CardHeader>
    <CardTitle> "Two-Factor Authentication"
  </CardHeader>
  <CardContent>
    <p> "Enter the 6-digit code from your authenticator app."
    <form>
      1. Single input: 6-digit numeric code
         - inputMode="numeric"
         - pattern="[0-9]{6}"
         - autoComplete="one-time-code"
         - className: text-center text-xl h-11 (44px per accessibility)
         - aria-label="6-digit verification code"
      2. "Verify" primary button (full width)
      3. "Back to login" link -> resets to Step 1
    </form>
    <p className="text-muted-foreground text-sm">
      "Code expires in {M}:{SS}" -- 5-minute countdown (per D-08)
    </p>
  </CardContent>
</Card>
```

5-minute countdown timer:
- Start a `useEffect` with `setInterval` when entering Step 2
- Count down from 300 seconds, format as M:SS
- On expiry: redirect to Step 1, show toast "Session expired. Please sign in again."
- `aria-live="off"` on the countdown (per UI-SPEC: not announced repeatedly)

On successful verify-2fa: redirect to `redirectAfterLogin` or `/`
On invalid code: show "Invalid code. Please try again." alert

**Addresses review concern (expired 2FA, page refresh during pending 2FA, locked-out state):**
- **Expired 2FA:** When verify-2fa returns 401 with "Session expired", reset to Step 1, show toast "Session expired. Please sign in again."
- **Page refresh during pending 2FA:** On page load, the component starts at Step 1 (credentials). There is no client-side persistence of the pending-2FA state. A page refresh during Step 2 effectively resets the flow. The pending-2FA session on the server will expire after 5 minutes anyway.
- **Locked-out state:** When login returns 429, show the lockout alert. Disable the form submit button and show countdown if rate limit headers (`Retry-After` or `X-RateLimit-Reset`) are present. Otherwise show static "Try again in 15 minutes."

Step transition: use `aria-live="polite"` region that announces "Two-Factor Authentication" when transitioning.

Read the redirect query param on mount: `const params = new URLSearchParams(location.search); const redirect = params.get('redirect');` Store in Zustand.
