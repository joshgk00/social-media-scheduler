`PUT /api/settings/preferences`:
- Validate body with `preferencesUpdateSchema`
- Update user columns: timezone, dateFormat, entriesPerPage, updatedAt

`PUT /api/settings/password`:
- Validate body with `passwordChangeSchema`
- Fetch user by `req.session.userId`
- Call `verifyPassword(user.passwordHash, currentPassword)` -- if false: return 401 `{ error: 'Current password is incorrect.' }`

- Call `invalidateOtherSessions(redis, req.sessionID)` (per D-23: kill other sessions, keep current)

`POST /api/settings/2fa/setup`:
- Fetch user, check `totpEnabled` is false. If already enabled: return 409 `{ error: '2FA is already enabled.' }`
- Call `generateTotpSecret(user.email)`
- Store secret in session: `req.session.pendingTotpSecret = secret` (per Research Pitfall 6: don't persist to DB before verification)
- Return `{ secret, uri }` (client renders QR code from URI)

`POST /api/settings/2fa/verify`:
- Validate body with `totpVerifySchema`
- Check `req.session.pendingTotpSecret` exists. If not: return 400 `{ error: 'No pending 2FA setup. Start the setup process first.' }`

- If invalid: return 401 `{ error: 'Invalid code. Make sure your authenticator time is synced.' }`
- If valid: update user `totpSecret = req.session.pendingTotpSecret`, `totpEnabled = true`, `updatedAt = new Date()`
- Clear `req.session.pendingTotpSecret`
- Return `{ success: true }`
