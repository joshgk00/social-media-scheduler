       Leave the `tag_search_vector` line as a plain nullable tsvector column (it is maintained by trigger, not GENERATED).

       Pitfall 1: the regconfig literal `'english'` is required to keep the expression IMMUTABLE. Do NOT use `to_tsvector("text" || ...)` without it — Postgres will reject the migration with `generation expression is not immutable`.

    4. APPEND the following hand-written SQL to the end of `0009_phase-11-snippets-fts-calendar.sql`. Each statement separated by `--> statement-breakpoint` per Drizzle convention. Source: RESEARCH §Pattern 1 lines 172-223 — copy verbatim, do not paraphrase:

       Statement A — refresh function (LANGUAGE sql):
           CREATE OR REPLACE FUNCTION refresh_post_tag_vector(p_post_id uuid)
           RETURNS void LANGUAGE sql AS $$
             UPDATE posts
             SET tag_search_vector = COALESCE(
               (SELECT to_tsvector('english', string_agg(t.name, ' '))
                FROM post_tags pt
                JOIN tags t ON t.id = pt.tag_id
                WHERE pt.post_id = p_post_id),
               ''::tsvector
             )
             WHERE id = p_post_id;
           $$;

       Statement B — plpgsql trigger function:
           CREATE OR REPLACE FUNCTION post_tags_refresh_vector()
           RETURNS trigger LANGUAGE plpgsql AS $$
           BEGIN
             IF TG_OP = 'DELETE' THEN
               PERFORM refresh_post_tag_vector(OLD.post_id);
               RETURN OLD;
             ELSE
               PERFORM refresh_post_tag_vector(NEW.post_id);
               RETURN NEW;
             END IF;
           END;
           $$;

       Statement C — drop existing trigger if present (idempotent re-apply):
           DROP TRIGGER IF EXISTS post_tags_after_change ON post_tags;

       Statement D — create trigger:
           CREATE TRIGGER post_tags_after_change
           AFTER INSERT OR DELETE ON post_tags
           FOR EACH ROW EXECUTE FUNCTION post_tags_refresh_vector();

       Statement E — one-time backfill (D-08: forward-only migration with one-pass tag-vector backfill):
           UPDATE posts SET tag_search_vector = COALESCE(
             (SELECT to_tsvector('english', string_agg(t.name, ' '))
              FROM post_tags pt JOIN tags t ON t.id = pt.tag_id
              WHERE pt.post_id = posts.id),
             ''::tsvector
           );

       Statement F — GIN index (D-07: combined-tsvector single GIN index):
           CREATE INDEX IF NOT EXISTS posts_fts_idx ON posts
             USING gin ((search_vector || tag_search_vector));

       `IF NOT EXISTS` on the GIN index protects against re-runs (the migration runner at `packages/db/src/migrate.ts` swallows duplicate-object SQLSTATE codes, but `IF NOT EXISTS` is more explicit). The trigger uses `DROP TRIGGER IF EXISTS` before `CREATE TRIGGER` for the same reason.
