## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API (session) | Authenticated via `requireAuth` + Redis-backed session; userId trusted to come from session, never from body |
| API → DB (snippet queries) | Every query filters by userId; cross-tenant queries impossible |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-05-01 | Information Disclosure (IDOR — read another user's snippet) | snippet.service.ts | mitigate | Every CRUD function does `where(and(eq(snippets.id, id), eq(snippets.userId, sessionUserId)))`; cross-user GET returns 404 (never 403 — no existence leak); integration test asserts another user's body never appears in response |
| T-11-05-02 | Tampering (CSRF on POST/PATCH/DELETE) | snippets routes | mitigate | Routes mounted AFTER existing csrf-csrf middleware in app.ts; inherits Phase 1 CSRF protection. No new exception |
| T-11-05-03 | Tampering (mass assignment via extra body fields) | createSnippetSchema, updateSnippetSchema | mitigate | Plan 01 schemas use `.strict()` (Zod rejects unknown keys); 400 on extra fields |
| T-11-05-04 | Input Validation (oversized name/body) | createSnippetSchema | mitigate | name max 100, body max 10_000 (plan 01); 400 on overflow |
| T-11-05-05 | Tampering (duplicate-name race past app-layer check) | unique constraint at DB | mitigate | DB-level `uniqueIndex(userId, lower(name))` (plan 03); race-safe; 23505 mapped to 409 in service |
| T-11-05-06 | Information Disclosure (stack traces / DB error leakage) | route handlers | mitigate | Service throws `SnippetServiceError(message, statusCode)`; route handler returns `{ error: err.message }` only — message is curated, no stack trace leaked |
</threat_model>
