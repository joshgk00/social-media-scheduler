<objective>
Wire `{{snippet:name}}` substitution into the CSV bulk-import worker handlers (both `scheduled` and `queue` variants — they share the same row-processing skeleton). Substitution runs once per job: load all snippets for `job.data.userId` into a `Map<lowercaseName, body>`, walk rows, call `substituteSnippetsInText` for each row's text. Rows whose tokens reference a missing snippet are added to the existing bulk-op error report (`{ rowNumber, message }` shape from Phase 10) and excluded from the insert batch.

Purpose: SNIP-03 acceptance — uploading a CSV with three rows (1 valid token, 1 missing token, 1 no token) produces 2 inserted posts with substituted text, 0 inserted posts containing `{{` or `}}`, and 1 error row in the bulk-op report.

Output: Modified handlers + integration test extension.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CSV row text → worker → DB | User-uploaded CSV text reaches the worker; substitution happens before INSERT |
| Snippet lookup → DB | Worker queries snippets owned by job.data.userId only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-08-01 | Information Disclosure (cross-tenant snippet substitution) | csv-import handler | mitigate | `where(eq(snippets.userId, job.data.userId))` filters snippet lookup by job owner; Test 5 asserts another user's snippet body never resolves |
| T-11-08-02 | Tampering (token injection — `{{snippet:'; DROP TABLE--}}`) | substituteSnippetsInText | mitigate | Token regex `[a-zA-Z0-9_\\- ]+` (plan 01) restricts the charset; SQL chars like `';` cannot match, regex returns no replacement and the literal token is preserved (which Postgres treats as plain text in the parameter-bound INSERT) |
| T-11-08-03 | Information Disclosure (snippet body XSS in stored post) | inserted post text | accept | Stored text is never rendered as HTML in this app; the eventual social-platform render is the only consumer and platform APIs handle their own escaping |
| T-11-08-04 | Logic flaw (rows with missing snippet partially inserted) | csv-import handler | mitigate | Failed rows are explicitly excluded from `validRows` via `continue`; integration test 3 asserts 2 inserts + 1 error row, never partial substitution |
| T-11-08-05 | Information Disclosure (full row content logged) | logger calls | mitigate | Per CLAUDE.md §Module Structure logger usage: log only `{ rowNumber, missingName }` — NEVER the full row body. Pino redact still applies via shared logger config |
| T-11-08-06 | DoS (oversized CSV with thousands of tokens) | substituteSnippetsInText | mitigate | MAX_SNIPPET_INPUT_LENGTH 50_000 (plan 01) short-circuits per-row text. CSV row size is already bounded by Phase 10 row schema |
</threat_model>
