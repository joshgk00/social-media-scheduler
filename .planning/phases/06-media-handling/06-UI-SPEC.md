# Phase 6 — UI Design Contract

> Visual and interaction contract for the Media Handling phase. Generated by gsd-ui-researcher. All values pre-populated from Phase 5 UI-SPEC (the canonical design baseline), CONTEXT.md D-01 through D-15, RESEARCH.md architecture patterns, and `packages/web/src/index.css`. Phase 6 replaces the placeholder media drop zone in NewPostPage.tsx (lines 322-327) with a fully functional upload and preview system, extends the settings page with a storage usage card, and adds transcoding status indicators.

---

## Phase UI Surfaces

Every surface below has a contract in the sections that follow.

| Surface | Scope | Kind |
|---------|-------|------|
| Media drop zone | Replace placeholder in NewPostPage.tsx with drag-and-drop upload zone + file picker | Extension of NewPostPage (and EditPostPage) |
| Media thumbnail grid | Sortable grid of uploaded media with progress overlays, status indicators, remove/reorder | New component integrated into post form |
| Transcoding status overlay | Spinner/error state on video thumbnails during async transcoding | New component (part of MediaThumbnail) |
| Storage usage card | Total storage, breakdown by type, file count on Settings page | New section on existing SettingsPage |
| Media in post lists | Media indicator icon/count on PostsPage and queue posts list rows | Extension of existing table rows |

## Spacing Scale

Phase 6 uses the Tailwind default spacing scale (established in Phase 2, continued through Phases 3-5). No new tokens.

| Token | Tailwind class | Value | Usage in Phase 6 |
|-------|----------------|-------|------------------|
| 1 | `p-1` / `gap-1` | 4px | Remove button (X) internal padding, thumbnail grid gap on mobile |
| 2 | `p-2` / `gap-2` | 8px | Thumbnail internal padding, status overlay padding, drop zone icon-to-text gap |
| 3 | `p-3` / `gap-3` | 12px | Thumbnail grid cells, storage usage card metric spacing |
| 4 | `p-4` / `gap-4` | 16px | Drop zone padding, storage card padding, form field gaps (existing) |
| 6 | `p-6` / `gap-6` | 24px | Drop zone vertical padding (tall zone), settings section gaps (existing) |
| 8 | `p-8` / `gap-8` | 32px | Drop zone empty state vertical padding (matches placeholder at line 323 of NewPostPage) |

Standard set: {4, 8, 12, 16, 24, 32}. **Declared exception:** 12px (`gap-3`) -- used for thumbnail grid cells where 16px wastes space in a dense grid. Still a multiple of 4. Consistent with Phase 4/5 exception. No other custom spacing values permitted in Phase 6.

### Accent reserved for (exhaustive list for Phase 6):

1. Drop zone border highlight on drag-over (`border-primary`)
2. Upload progress bar fill color (inside `Progress` component)

### Destructive reserved for:

1. Remove media button (X) on each thumbnail -- icon color `text-destructive` on hover
2. Failed transcode error overlay background (`bg-destructive/10`) and error text (`text-destructive`)
3. Client-side validation error text (`text-destructive`) for rejected files

### Success reserved for:

1. Completed transcode status indicator (green checkmark overlay on video thumbnail)
2. Upload complete state (brief green check before overlay fades)

### Warning reserved for:

1. "Transcoding..." spinner overlay text (`text-warning`) on video thumbnails during processing

### Transcode status mapping

| Transcode status | Visual treatment | Color role |
|------------------|-----------------|------------|

## Interaction States

### Media Drop Zone (`MediaDropZone.tsx`)

Replaces the placeholder `<div>` at lines 322-327 of `NewPostPage.tsx`. Also used in `EditPostPage.tsx`.

| State | Visual treatment |
|-------|-----------------|
| Idle (no files) | Dashed border (`border-2 border-dashed border-border`), `rounded-lg`, `bg-card/50`, centered `Upload` icon (lucide) at 32px + two lines of text. Full height: `p-8` padding. Clickable (opens native file picker). |
| Idle (has files, under limit) | Drop zone shrinks to a compact strip: single row, `p-4`, `Upload` icon at 16px + "Add more files" text. Still clickable and droppable. |
| Idle (at file limit) | Drop zone hidden entirely. Thumbnail grid is the only visible element. |
| Drag-over | Border transitions to `border-primary border-solid`, background `bg-primary/5`. Text changes to "Drop to upload". |
| Drag-over (invalid file type) | Border `border-destructive border-solid`, background `bg-destructive/5`. Text: "Unsupported file type". |
| Disabled (post in `publishing` state on edit page) | Entire drop zone hidden. Thumbnails shown read-only (no X, no drag handles). |

Input attributes:
- `accept`: Derived from platform limits (e.g., `image/jpeg,image/png,image/gif,image/webp,video/mp4,video/quicktime` for Twitter)
- `multiple`: `true` (up to platform limit)
- Hidden `<input type="file">` triggered by click on zone or "Add more files" strip

### Media Thumbnail Grid (`MediaThumbnailGrid.tsx`)

Appears directly below the drop zone (or replaces it when at file limit). Uses `@dnd-kit/sortable` with `rectSortingStrategy` for grid-based drag reorder.

Layout:
- Grid: `grid grid-cols-2 gap-3` on mobile, `grid-cols-4 gap-3` on desktop (`md:` breakpoint)
- When 1 file: single thumbnail, full first-column width
- When 2 files: 2 columns
- When 3+ files: fills grid left-to-right, top-to-bottom

Each thumbnail cell:
- `aspect-square rounded-md overflow-hidden relative bg-card`
- Image: `object-cover w-full h-full` using the thumbnail URL (300px version from server)
- Video: generic video icon centered (`Film` lucide icon at 32px, `text-muted-foreground`) with filename below
- Remove button: absolute positioned top-right, `p-1` circle, `bg-background/80 backdrop-blur-sm`, `X` icon at 14px. On hover: `text-destructive`.
- Drag handle: absolute positioned top-left, `GripVertical` icon at 14px, `text-muted-foreground`, `cursor-grab`. Only shown when more than 1 file.

| State | Visual treatment |
|-------|-----------------|
| Uploading | Thumbnail shows a semi-transparent overlay (`bg-background/60`). `Progress` bar (shadcn) centered vertically. Percentage text (`text-xs`) below progress bar. No remove button visible during upload. No drag handle. |
| Upload complete (image) | Overlay fades out. Thumbnail visible. Remove button and drag handle visible. |
| Upload complete (video, transcoding) | Overlay shows spinner (`Loader2` at 24px, animate-spin) + "Transcoding..." text (`text-xs text-warning`). Remove button visible. No drag handle (can't reorder during transcode). |
| Transcode queued | Overlay with `Loader2` at 24px (not spinning, static) + "Queued..." text (`text-xs text-muted-foreground`). |
| Transcode complete | Green check overlay (`CheckCircle` at 24px, `text-success`) for 1.5 seconds, then fades to show video thumbnail (or generic video icon). Drag handle appears. |
| Transcode failed | Red overlay (`bg-destructive/10`). `AlertCircle` icon at 24px `text-destructive`. Error text `text-xs text-destructive` below icon. Two links below: "Retry" (`text-primary cursor-pointer`) and "Remove" (`text-destructive cursor-pointer`). |
| Drag active | Dragged item gets `opacity-50 ring-2 ring-primary`. Drop placeholder shows `border-2 border-dashed border-primary`. |
| Drag over other items | Items shift smoothly via @dnd-kit transition. |
| Read-only (publishing state) | No remove buttons, no drag handles, no drop zone. Thumbnails display with slight `opacity-80`. |

### Transcoding Status Polling

- Poll `GET /api/media/:id/status` every 3 seconds while any media has `transcodeStatus === 'processing'` or `'pending'`
- Use TanStack Query `refetchInterval: 3000` with `enabled` guard: only poll when pending/processing media exists
- Pause polling when `document.hidden` (tab backgrounded)
- Stop polling when all media reach `completed`, `failed`, or `not_applicable`

### Submit Button Interaction with Transcoding

| Condition | Button state |
|-----------|-------------|
| Any media has `transcodeStatus === 'processing'` or `'pending'` | Button disabled. Tooltip: "Video is still transcoding." |
| Any media has `transcodeStatus === 'failed'` | Button disabled. Tooltip: "Fix or remove failed media before submitting." |
| All media `completed` or `not_applicable` | Button enabled (normal behavior) |
| Post saved as draft | Allowed even during transcoding. Draft save is always available. |

### Storage Usage Card (`StorageUsageCard.tsx`)

Added as a new section on the Settings page, positioned after `SecuritySection` (last section).

Layout: `Card` component with `CardHeader` + `CardContent`. Same width constraint as existing sections (`max-w-[640px]`).

| Element | Layout |
|---------|--------|
| Card title | "Storage Usage" in `CardHeader` with `CardTitle` (text-lg font-semibold) |
| Metrics row | Three metrics side by side in a `grid grid-cols-3 gap-4` |
| Each metric | Stacked: value on top (`text-2xl font-semibold`), label below (`text-xs text-muted-foreground`) |
| Images metric | Value: formatted size string. Label: `Images ({count} files)` |
| Videos metric | Value: formatted size string. Label: `Videos ({count} files)` |
| Total metric | Value: formatted size string. Label: `Total` |

Size formatting: `0 B`, `145 MB`, `2.3 GB` -- use 1 decimal place for GB, whole numbers for MB and below. Implementation uses a simple `formatBytes()` utility.

| State | Visual treatment |
|-------|-----------------|
| Loading | Three `Skeleton` rectangles in the metrics grid (`h-16 rounded-md`). |
| Loaded | Metrics displayed as described. |
| Empty (no media) | Single line of text: "No media uploaded yet." (`text-sm text-muted-foreground`, centered). |
| Error | Single line: "Couldn't load storage info." (`text-sm text-destructive`). |

### Media Indicator in Post Lists

Extend existing PostsPage and queue posts list table rows:

| Change | Detail |
|--------|--------|
| New column indicator | Not a full column. Instead, add a small `Image` icon (lucide, 14px) next to the post text preview when the post has attached media. Shows count: `Image` icon + `{count}` in `text-xs text-muted-foreground`. |
| Tooltip | Hover on media indicator shows: "{count} image(s)" or "1 video" or "{count} image(s), 1 video (transcoding)". |
| Transcoding indicator | If any media is still transcoding, show a tiny `Loader2` icon (12px, animate-spin) next to the media count instead of the static `Image` icon. |

### Edit Post Media Handling

When editing a post (`EditPostPage.tsx`):

| Scenario | Behavior |
|----------|----------|
| Post has existing media | Thumbnails pre-populated in the grid from server data. All existing interactions (remove, reorder, add more) available. |
| User removes media | Media soft-deleted via `DELETE /api/media/:id`. Thumbnail removed from grid immediately (optimistic). |
| User adds new media | Same upload flow as new post. New files added to the grid alongside existing ones. |
| User reorders | Sort order updated. New order sent with post save. |
| Post in `publishing` state | Media displayed read-only. No drop zone, no remove buttons, no drag handles. (STATE-02 enforcement.) |

## Accessibility Contract

All Phase 6 UI inherits the accessibility baseline from `packages/web/CLAUDE.md`: WCAG AA contrast (4.5:1 text, 3:1 large), `<main>` landmark on every page, semantic HTML before divs, labeled inputs, visible focus rings via `--color-ring`.

| Requirement | Detail |
|-------------|--------|
| Drop zone keyboard access | The hidden `<input type="file">` is focusable. The drop zone wrapper has `role="button"` and `tabIndex={0}`. Pressing Enter or Space opens the file picker. `aria-label="Upload media files"`. |
| Drop zone drag feedback | Drag-over state announced via `aria-live="polite"` region: "File ready to drop" / "Unsupported file type". |
| Thumbnail grid | Each thumbnail has `aria-label="{filename}, {status}"` (e.g., "photo.jpg, uploaded" or "video.mp4, transcoding"). |
| Remove button | `aria-label="Remove {filename}"` on each X button. |
| Drag handle | `aria-label="Reorder {filename}"`. @dnd-kit provides `aria-roledescription="sortable"` and keyboard-based reorder (Space to pick up, arrow keys to move, Space to drop). |
| Upload progress | Progress bar has `aria-label="Uploading {filename}"` and `aria-valuenow={percent}`. |
| Transcoding status | Status text is visible (not color-only). Screen readers receive status updates via `aria-live="polite"` region in the thumbnail. |
| Failed transcode | "Retry" and "Remove" links are focusable `<button>` elements with descriptive text. Error message is the accessible description of the thumbnail. |
| Submit disabled reason | Disabled submit button has `aria-describedby` pointing to a visually hidden element explaining why (e.g., "Video is still transcoding."). Tooltip provides the same info for pointer users. |
| Storage usage card | Metrics use `aria-label` for full context: `aria-label="Images: 145 MB, 23 files"`. |
| Media indicator in lists | Icon + count text provides accessible name. Transcoding spinner has `aria-label="Media transcoding in progress"`. |
