    **1. Transcode service** (`packages/worker/src/transcode.service.ts`):
    ```typescript
    import { spawn } from 'node:child_process';

    const TRANSCODE_TIMEOUT_MS = 300_000; // 5 minutes (MEDIA-04, D-09)

    export function transcodeVideo(inputPath: string, outputPath: string): Promise<void> {
      return new Promise((resolve, reject) => {
        const args = [
          '-i', inputPath,
          '-vf', 'scale=-2:720',           // 720p, keep aspect ratio (D-06)
          '-c:v', 'libx264',               // H.264 codec
          '-preset', 'fast',               // Speed/compression balance
          '-crf', '23',                     // Quality (23 is default/balanced)
          '-c:a', 'aac',                    // AAC audio
          '-b:a', '128k',                   // 128kbps audio bitrate
          '-movflags', '+faststart',        // Web-optimized MP4
          '-y',                             // Overwrite output
          outputPath,
        ];

        const proc = spawn('ffmpeg', args);
        const timeout = setTimeout(() => {
          proc.kill('SIGKILL');
          reject(new Error('Transcoding timeout exceeded (5 minutes)'));
        }, TRANSCODE_TIMEOUT_MS);

        let stderr = '';
        proc.stderr.on('data', (chunk: Buffer) => { stderr += chunk.toString(); });

        proc.on('close', (code) => {
          clearTimeout(timeout);
          if (code === 0) resolve();
          else reject(new Error(`ffmpeg exited with code ${code}: ${stderr.slice(-500)}`));
        });

        proc.on('error', (err) => {
          clearTimeout(timeout);
          reject(err);
        });
      });
    }
    ```

    **2. Transcode worker** (`packages/worker/src/transcode-worker.ts`):
    Factory function `createTranscodeWorker(deps: { redis: Redis, db: WorkerDb, storage: StorageBackend })`:
    - Creates `new Worker(QUEUE_NAMES.transcode, processor, { connection: redis, concurrency: 1, lockDuration: 360_000 })`
    - `lockDuration: 360_000` (6 minutes) is > timeout (5 min) to prevent stalled detection during transcode
    - Job data shape: `{ mediaId: string, inputKey: string, profileId: string }`
    - Processor logic:
      1. Update `post_media` set `transcode_status = 'processing'` where id = mediaId
      2. Download original from storage to temp file: `const inputPath = path.join(os.tmpdir(), randomUUID() + '_input')`, write `await storage.get(inputKey)` to inputPath
      3. Generate output temp path: `const outputPath = path.join(os.tmpdir(), randomUUID() + '.mp4')`
      4. Call `await transcodeVideo(inputPath, outputPath)`
      5. On success:
         - Read output file stats for fileSize
         - Generate output storage key: `media/${profileId}/${year}/${month}/${randomUUID()}.mp4`
         - Save to storage: `await storage.save(outputKey, fs.createReadStream(outputPath), 'video/mp4')`
         - Update `post_media`: set `filePath = outputKey, fileSize = stats.size, mimeType = 'video/mp4', transcodeStatus = 'completed', transcodeError = null`
      6. On failure (catch):
         - Update `post_media`: set `transcodeStatus = 'failed', transcodeError = err.message.slice(0, 1000)`
         - Rethrow so BullMQ marks the job as failed
      7. Finally: clean up both temp files with unlink, ignore ENOENT
    - Worker events: `worker.on('failed', (job, err) => logger.error(...))`, `worker.on('completed', (job) => logger.info(...))`
    - Return the worker instance for shutdown management
