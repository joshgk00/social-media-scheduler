    **1. Media cleanup worker** (`packages/worker/src/media-cleanup-worker.ts`):

    `createMediaCleanupWorker(deps: { redis: Redis, db: WorkerDb, storage: StorageBackend })`:
    - Creates `new Worker(QUEUE_NAMES.mediaCleanup, processor, { connection: redis, concurrency: 1 })`
    - Processor logic:
      1. **Permanent deletion of soft-deleted files > 30 days:**
         ```typescript
         const thirtyDaysAgo = new Date(Date.now() - 30 * 24 * 60 * 60 * 1000);
         const expiredMedia = await db.select()
           .from(postMedia)
           .where(and(
             isNotNull(postMedia.deletedAt),
             lt(postMedia.deletedAt, thirtyDaysAgo),
           ));
         ```
         For each row:
         - Try `storage.delete(row.filePath)` -- catch and log on error, continue
         - If `row.thumbnailPath`: try `storage.delete(row.thumbnailPath)` -- catch and log on error, continue
         - Hard-delete the database row: `db.delete(postMedia).where(eq(postMedia.id, row.id))`
         - Log: `{ mediaId: row.id, filePath: row.filePath }, 'Permanently deleted soft-deleted media'`

      2. **Orphan cleanup > 24 hours:**
         ```typescript
         const twentyFourHoursAgo = new Date(Date.now() - 24 * 60 * 60 * 1000);
         const orphans = await db.select()
           .from(postMedia)
           .where(and(
             isNull(postMedia.postId),
             isNull(postMedia.deletedAt),
             lt(postMedia.createdAt, twentyFourHoursAgo),
           ));
         ```
         For each orphan: same storage.delete + hard-delete pattern as above.
         Log: `{ orphanCount: orphans.length }, 'Cleaned up orphaned uploads'`

      3. Log summary: `{ deletedExpired: expiredMedia.length, deletedOrphans: orphans.length }, 'Media cleanup job completed'`

    - Return the worker instance

|-----------|----------|-----------|-------------|-----------------|
| T-06-16 | Denial of Service | Cleanup job deleting active files | mitigate | 30-day grace period for soft-deleted files; 24-hour grace for orphans; both generous enough to prevent accidental data loss |
| T-06-17 | Information Disclosure | Storage usage exposes file counts | accept | Single-user app; storage usage is the user's own data; endpoint requires authentication |
