    **1. StorageBackend interface** (`packages/shared/src/storage/storage-backend.ts`):
    ```typescript
    export interface StorageBackend {
      save(key: string, data: Buffer | NodeJS.ReadableStream, contentType: string): Promise<void>;
      get(key: string): Promise<Buffer>;
      delete(key: string): Promise<void>;
      getUrl(key: string): string;
      exists(key: string): Promise<boolean>;
    }
    ```

    **2. LocalStorage** (`packages/shared/src/storage/local-storage.ts`):
    - Constructor takes `rootDir: string` (defaults to `./data/media`)
    - `save()`: resolve key against rootDir, validate with `path.resolve() + startsWith(rootDir + path.sep)` guard (per D-10, reuse avatar upload path traversal pattern), create parent directories with `mkdir -p`, write file via `fs.promises.writeFile`
    - `get()`: resolve + guard, read via `fs.promises.readFile`
    - `delete()`: resolve + guard, `fs.promises.unlink`, ignore ENOENT
    - `getUrl()`: return `/media/${key}` (served by express.static in API)
    - `exists()`: resolve + guard, `fs.promises.access` returns true, ENOENT returns false

    **3. S3Storage** (`packages/shared/src/storage/s3-storage.ts`):
    - Constructor takes `{ endpoint: string, bucket: string, accessKey: string, secretKey: string, region?: string }`
    - Creates `S3Client` from `@aws-sdk/client-s3` with `forcePathStyle: true` (per RESEARCH.md Pitfall 7 for MinIO/self-hosted compatibility), `region: config.region || 'us-east-1'`
    - `save()`: `PutObjectCommand({ Bucket, Key: key, Body: data, ContentType: contentType })`
    - `get()`: `GetObjectCommand({ Bucket, Key })`, collect stream into Buffer
    - `delete()`: `DeleteObjectCommand({ Bucket, Key })`
    - `getUrl()`: return `${endpoint}/${bucket}/${key}` (direct URL for non-private buckets)
    - `exists()`: `HeadObjectCommand({ Bucket, Key })`, return true, catch NotFound return false

    **4. Factory** (`packages/shared/src/storage/index.ts`):
    ```typescript
    export function createStorageBackend(): StorageBackend {
      const backend = process.env.MEDIA_STORAGE_BACKEND || 'local';
      if (backend === 's3') {
        return new S3Storage({
          endpoint: requireEnv('S3_ENDPOINT'),
          bucket: requireEnv('S3_BUCKET'),
          accessKey: requireEnv('S3_ACCESS_KEY'),
          secretKey: requireEnv('S3_SECRET_KEY'),
        });
      }
      const mediaDir = process.env.MEDIA_DIR || './data/media';
      return new LocalStorage(mediaDir);
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| StorageBackend key input | Keys derived from UUID + profile ID; never from user-supplied filenames |
| S3 endpoint config | Admin-configured env var; not user-supplied (no SSRF risk from user input) |
| Local filesystem access | Path traversal guard on all LocalStorage operations |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | LocalStorage key resolution | mitigate | `path.resolve()` + `startsWith(rootDir + path.sep)` guard rejects keys containing `../` or absolute paths |
| T-06-02 | Information Disclosure | S3Storage.getUrl() | accept | URLs are direct S3 paths; single-user app with no public access. Presigned URLs can be added later if S3 bucket is private. |
| T-06-03 | Denial of Service | S3 endpoint misconfiguration | accept | S3_ENDPOINT is admin-configured via env var; misconfiguration causes upload failure, not security breach |
</threat_model>
