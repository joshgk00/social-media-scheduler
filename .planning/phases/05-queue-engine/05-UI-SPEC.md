# Phase 5 — UI Design Contract

> Visual and interaction contract for the Queue Engine phase. Generated by gsd-ui-researcher. All values pre-populated from Phase 4 UI-SPEC (the canonical design baseline), CONTEXT.md D-01 through D-14, RESEARCH.md architecture patterns, and `packages/web/src/index.css`. Phase 5 introduces no new design primitives -- it extends the existing system with new pages and components following established patterns.

## Spacing Scale

Phase 5 uses the Tailwind default spacing scale (established in Phase 2, continued through Phase 3/4). No new tokens.

| Token | Tailwind class | Value | Usage in Phase 5 |
|-------|----------------|-------|------------------|
| 1 | `p-1` / `gap-1` | 4px | Icon-to-text inline gaps (lucide icon + label), reorder button internal padding |
| 2 | `p-2` / `gap-2` | 8px | Compact stacks inside schedule builder rows, badge internal padding, reorder button spacing |
| 3 | `p-3` / `gap-3` | 12px | Within-group spacing inside schedule builder sections (hour slot grid cells) |
| 4 | `p-4` / `gap-4` | 16px | Default element spacing, dialog body padding, card padding, form field gaps |
| 6 | `p-6` / `gap-6` | 24px | Section padding inside PageLayout, dialog header padding, schedule builder section gaps |
| 8 | `p-8` / `gap-8` | 32px | Major page-level section breaks |
| 12 | `py-12` | 48px | Vertical padding on empty-state blocks |

Standard set: {4, 8, 12, 16, 24, 32, 48}. **Declared exception:** 12px (`p-3`/`gap-3`) -- used only for hour slot grid cell spacing inside the schedule builder where 16px wastes too much space for a dense checkbox grid. Still a multiple of 4. Matches Phase 4's exception justification. No other custom spacing values permitted in Phase 5.

### Accent reserved for (exhaustive list for Phase 5):

1. Primary CTA button on Queue creation/edit form save button ("Create Queue" / "Save Queue")
2. "Save to Queue" button on the queue post creation form
3. "Add Post" button on the queue posts page

Nothing else uses accent. Status badges and semantic states use their own tokens.

### Destructive reserved for:

1. "Delete Queue" menu item text in the per-queue kebab
2. "Delete Queue" AlertDialog confirm button
3. "Delete Post" menu item text in queue posts kebab (existing pattern)
4. `auto_destructing` status badge (subtle destructive tint -- same as `failed` but semantically different; see badge mapping)

### Success reserved for:

1. `published` status badge in queue posts list (existing pattern)
2. `destroyed` status badge (post successfully auto-destructed -- green tint, because destruction was the intended outcome)

### Warning reserved for:

1. Seasonal window "paused" indicator on queue list (queue is outside its seasonal date range)
2. Queue `isPaused` badge on queue list (manually paused)

### Status badge mapping

Post status badges in queue posts list follow the same mapping as Phase 4:

| Post status | Badge variant | Color role |
|-------------|---------------|------------|
| `queued` | `outline` | foreground + border |
| `publishing` | `secondary` (with spinner icon) | muted |
| `published` | custom success | success |
| `failed` | `destructive` | destructive |
| `auto_destructing` | custom destructive tint | destructive/muted |
| `destroyed` | custom success tint | success (muted) |

Queue-level status indicators:

| Queue state | Badge variant | Color role |
|-------------|---------------|------------|
| Active | `outline` with green dot | success |
| Paused | `secondary` with amber dot | warning |
| Seasonal pause | `secondary` with amber dot + clock icon | warning |
| Empty (no posts) | `secondary` | muted |

### Queue create/edit page (`/queues/new`, `/queues/:id/edit`)

Layout: single-column form inside the standard page layout (`p-6 lg:p-8`). Mirrors the general structure of NewPostPage (top-to-bottom form fields, save/cancel at the bottom).

**Schedule builder** is the primary visual element -- a section containing:

1. **Interval configuration** -- interval type radio/select + value + unit in a single row
2. **Days of week** -- 7 checkboxes in a horizontal row (Sun through Sat), each labeled with the 3-letter abbreviation (Sun, Mon, Tue, Wed, Thu, Fri, Sat)
3. **Hour windows** -- grid of checkboxes from 6am to 11pm (18 slots). Laid out as a 6-column grid on desktop, 3-column on mobile. Each shows the hour in the user's preferred format (12h or 24h based on settings). Include "Select All" / "Clear All" links above the grid.
4. **Start date** -- date picker input (using native `<input type="date">`)
5. **Seasonal window** -- two date inputs (start MM-DD, end MM-DD) + "Repeat annually" toggle. Collapsed by default, expandable via a "Add seasonal window" link.

| State | Behavior |
|-------|----------|
| Loading (edit mode) | Skeleton form fields while queue data loads. |
| Saving | Save button shows `Loader2` spinner; form disabled. |
| Success (create) | Toast; navigate to `/queues/:id/posts`. |
| Success (edit) | Toast; navigate to `/queues/:id/posts`. |
| Error | Toast; form re-enables; save button re-enables. |
| Validation errors | Inline `FormMessage` below each invalid field (existing shadcn form pattern). |

## Accessibility Contract

All Phase 5 UI inherits the accessibility baseline from `packages/web/CLAUDE.md`: WCAG AA contrast (4.5:1 text, 3:1 large), `<main>` landmark on every page, semantic HTML before divs, labeled inputs, visible focus rings via `--color-ring`.

| Requirement | Detail |
|-------------|--------|
| Page landmarks | Queue list, create/edit, and posts pages all wrap content in `<main>`. |
| Dialog focus management | Spinnable Variants, Delete Queue, and Delete Post dialogs must trap focus and restore focus to trigger on close -- Radix handles this by default. |
| Dialog labeling | Every `Dialog` and `AlertDialog` uses `DialogTitle` + `DialogDescription`. No empty titles. |
| Reorder buttons | Each up/down button must have `aria-label`: "Move post to position {N-1}" / "Move post to position {N+1}". Disabled buttons include `aria-disabled="true"`. |
| Hour window checkboxes | Each checkbox must be associated with a visible `<label>` showing the hour (e.g., "9 AM"). The fieldset wrapping the grid uses `<fieldset>` + `<legend>` reading "Hour windows". |
| Day-of-week checkboxes | Wrapped in `<fieldset>` + `<legend>` reading "Days of week". Each checkbox labeled with full day name via `aria-label` (visible text is 3-letter abbreviation). |
| Status badges | Badge text is always present alongside color -- never color-coded without text label. |
| Queue name links | Queue name in the list is a standard `<a>` / `NavLink` -- no `onClick` handlers on `<div>` or `<span>`. |
| Toast notifications | `sonner` configured with `aria-live="polite"` (default -- verify). |
| Keyboard navigation | All kebab menu items, reorder buttons, form controls, and dialog controls must be keyboard-navigable. |
| Cursor indicator | The "next to publish" left border highlight is supplemented with a `text-xs` label "Next" in the Position column for screen reader users, using `aria-label="Position {N} (next to publish)"`. |
