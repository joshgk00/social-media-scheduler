    Create `packages/shared/src/lib/schedule-evaluation.ts` with these pure functions using Luxon DateTime:

    1. `isWithinHourWindow(hourSlots: number[], userTimezone: string, now?: DateTime): boolean`
       - Convert now to user's timezone via `(now ?? DateTime.utc()).setZone(userTimezone)`
       - Return `hourSlots.includes(localNow.hour)`
       - Per RESEARCH.md Pitfall 3: during spring-forward, hour 2 naturally skips because localNow.hour jumps from 1 to 3

    2. `isDayOfWeekAllowed(daysOfWeek: number[], userTimezone: string, now?: DateTime): boolean`
       - Convert to local timezone
       - Map Luxon weekday (1=Mon..7=Sun) to SocialOomph convention (0=Sun..6=Sat): `const dow = localNow.weekday === 7 ? 0 : localNow.weekday;`
       - Return `daysOfWeek.includes(dow)`

    3. `hasIntervalElapsed(intervalType: 'fixed' | 'variable', intervalValue: number, intervalUnit: string, lastPublishedAt: DateTime | null, userTimezone: string, now?: DateTime): boolean`
       - If `lastPublishedAt` is null, return true (first publish ever)
       - For **fixed** intervals: align to midnight in user's timezone. Check if the current time slot (floor of now to interval boundary) is different from the slot of lastPublishedAt. Use Luxon `.startOf('day')` + interval arithmetic.
       - For **variable** intervals: compute elapsed time since last publish: `localNow.diff(localLast, intervalUnit).as(intervalUnit) >= intervalValue`

    4. `isWithinSeasonalWindow(seasonalStart: string | null, seasonalEnd: string | null, seasonalRepeat: boolean, now?: DateTime): boolean`
       - If both null, return true (no seasonal restriction)
       - Parse MM-DD strings to month and day integers
       - For non-repeating: compare current year's date against the range. If the queue was created for a specific year window, check against that year only.
       - For repeating (annual): check if current MM-DD falls within start-end. Handle cross-year wrapping: if start > end (e.g., '12-01' to '01-31'), the window wraps across year boundary -- current date is within if `currentMMDD >= start OR currentMMDD <= end`. (RESEARCH.md Assumption A2 mitigation)
