    ```typescript
    import { pgTable, uuid, integer, timestamp, text, varchar, index, pgEnum } from 'drizzle-orm/pg-core';
    import { sql } from 'drizzle-orm';
    import { posts } from './posts.js';

    export const postAttemptOutcome = pgEnum('post_attempt_outcome', [
      'success',
      'transient_fail',
      'permanent_fail',
      'cancelled',
    ]);

    export const postAttempts = pgTable(
      'post_attempts',
      {
        id: uuid('id').primaryKey().default(sql`gen_random_uuid()`),
        postId: uuid('post_id')
          .notNull()
          .references(() => posts.id, { onDelete: 'cascade' }),
        attemptNum: integer('attempt_num').notNull(),
        startedAt: timestamp('started_at', { withTimezone: true }).notNull(),
        finishedAt: timestamp('finished_at', { withTimezone: true }),
        outcome: postAttemptOutcome('outcome').notNull(),
        httpStatus: integer('http_status'),
        errorCode: text('error_code'),
        errorMessage: text('error_message'),
        platformPostId: varchar('platform_post_id', { length: 255 }),
      },
      (table) => ({
        postIdStartedAtIdx: index('post_attempts_post_started_idx').on(
          table.postId,
          table.startedAt,
        ),
      }),
    );

    export type PostAttempt = typeof postAttempts.$inferSelect;
    export type NewPostAttempt = typeof postAttempts.$inferInsert;
    ```

    Do NOT add any columns not listed in D-16. Do NOT add an `updatedAt` column — D-19 requires compact rows.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | `post_attempts` row insert without post ownership check | mitigate | Cascade FK to posts.id means deleting a post removes its history; ownership checks are enforced in the API/service layer (Plan 04 route handlers), not at the DB — documented so Plans 03/04 cannot skip the check |
| T-04-01-02 | Information Disclosure | `error_message` text column could accidentally persist token fragments if upstream classifier is buggy | mitigate | Schema allows NULL; Plan 02 error classifier is the control point. Schema comment added in a follow-up if we observe leakage. Low severity — the classifier reads from `ApiResponseError.errors[0].message` which does not echo request auth headers. |
| T-04-01-03 | Denial of Service | Unbounded growth of `post_attempts` with no retention job | accept | D-18 explicitly accepts retain-forever. 500 posts/month × ~200 bytes × 4 attempts ≈ 400KB/month — negligible on Proxmox SSD for v1. |
| T-04-01-04 | Tampering | New `monthlyTweetBudget` column has no DB CHECK constraint | mitigate | Zod schema in Plan 02 (`rate-limit.schema.ts`) enforces 1 ≤ budget ≤ 10000, 1 ≤ warn ≤ 99 before any INSERT/UPDATE reaches the DB. Acceptable split because only the API can write the column. |
| T-04-01-05 | Elevation of Privilege | Migration ran with superuser instead of app role | mitigate | Drizzle-kit uses the same `DATABASE_URL` the app uses. No separate superuser step. Verified by the existing Phase 1 migration pattern. |
