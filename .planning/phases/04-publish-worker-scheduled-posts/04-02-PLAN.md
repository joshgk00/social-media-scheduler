<objective>
Build the shared primitives and API-side services that the worker (Plan 03) and route handlers (Plan 04) will consume: queue name constants, error classifier, publish queue factory, **pure rate-limit calculators in @sms/shared**, DB-backed rate-limit wrapper service in @sms/api, and Zod schemas for rate limit config. Also adds the new npm dependencies (`bullmq`, `luxon`, `@bull-board/api`, `@bull-board/express`) to the api package and `luxon` + `twitter-api-v2` + `drizzle-orm` + `postgres` to the worker package.

**Architecture note (revision):** The rate-limit math (`checkTwitterBudget`, `checkBulkBudget`) lives in `@sms/shared/rate-limit/check-budget.ts` as pure functions that take `{ currentUsage, monthlyBudget, warnThresholdPercent, additionalCount }` and return `{ wouldExceed, warnThresholdHit, blockThresholdHit, remainingBudget, ... }`. The API-side `rate-limit.service.ts` is a thin wrapper that reads `currentUsage` from the DB and calls the shared pure function. The worker (Plan 03) imports the same pure function from `@sms/shared` directly — it reads its own usage count via the `WorkerDb` and calls the shared calculator. **No cross-package dependency between @sms/worker and @sms/api.**

LIMIT-05 (CSV bulk upload pre-flight) is satisfied by exporting `checkBulkBudget(args)` from `@sms/shared`. The Phase 10 CSV import flow will import this directly. Phase 4 ships the contract; Phase 10 wires the consumer.

Purpose: Everything downstream (worker, endpoints, UI fetches) depends on these building blocks. Writing them once here keeps Plans 03 and 04 focused on wiring.
Output: Five shared files (queues, error-classifier, check-budget, rate-limit schemas, post-history schemas) + DB-backed wrapper service in @sms/api + publish queue factory + dependencies added to package.json files.

    **3. `packages/shared/src/rate-limit/check-budget.ts`** — NEW FILE per revision Blocker 4. Pure rate-limit calculators usable from any package without DB access:

    ```typescript
    /**
     * Pure rate-limit calculators. Take a snapshot of usage + budget and return
     * the projection. No DB access, no time math beyond what the caller passes in.
     *
     * Used by:
     *   - @sms/api rate-limit.service (DB-backed wrapper)
     *   - @sms/worker post-lifecycle.service (runtime re-check, D-26)
     *   - @sms/api Phase 10 CSV bulk import (LIMIT-05) — wires `checkBulkBudget`
     */

    export interface BudgetCheckArgs {
      /** Number of posts already counted toward this billing cycle. */
      currentUsage: number;
      /** Profile's monthly budget cap (D-22, default 500). */
      monthlyBudget: number;
      /** Percent at which we surface a warning (D-22, default 80, range 1-99). */
      warnThresholdPercent: number;
      /** How many additional posts the caller wants to add (1 for single post, N for bulk). */
      additionalCount: number;
    }

    export interface BudgetCheckResult {
      currentUsage: number;
      budget: number;
      warnThresholdPercent: number;
      projectedCount: number;
      wouldExceed: boolean;
      warnThresholdHit: boolean;
      blockThresholdHit: boolean;
      /** Posts remaining BEFORE the additional batch is applied. Floors at 0. */
      remainingBudget: number;
    }

    export function checkTwitterBudget(args: BudgetCheckArgs): BudgetCheckResult {
      const projectedCount = args.currentUsage + args.additionalCount;
      const warnThreshold = Math.floor(
        args.monthlyBudget * (args.warnThresholdPercent / 100),
      );
      return {
        currentUsage: args.currentUsage,
        budget: args.monthlyBudget,
        warnThresholdPercent: args.warnThresholdPercent,
        projectedCount,
        wouldExceed: projectedCount > args.monthlyBudget,
        warnThresholdHit: projectedCount >= warnThreshold,
        blockThresholdHit: projectedCount >= args.monthlyBudget,
        remainingBudget: Math.max(0, args.monthlyBudget - args.currentUsage),
      };
    }

    /**
     * Semantic alias for CSV bulk upload pre-flight (LIMIT-05). The math is
     * identical to checkTwitterBudget — the alias exists so Phase 10's CSV
     * importer has a clear contract to wire against. Phase 10 will load
     * currentUsage from the DB, then call this with additionalCount = csvRowCount.
     */
    export function checkBulkBudget(args: BudgetCheckArgs): BudgetCheckResult {
      return checkTwitterBudget(args);
    }
    ```

    ```typescript
    import { DateTime } from 'luxon';
    import { sql, and, eq, gte, inArray } from 'drizzle-orm';
    import type { Db } from '@sms/db';
    import { posts, socialProfiles } from '@sms/db';
    import {
      checkTwitterBudget,
      checkBulkBudget,
      type BudgetCheckResult,
    } from '@sms/shared';

    const COUNTED_STATUSES = ['published', 'auto_destructing', 'destroyed'] as const;

    export interface UsageSnapshot {
      currentUsage: number;
      monthlyBudget: number;
      warnThresholdPercent: number;
      monthStartUtc: Date;
    }

    /**
     * Loads the live monthly usage counter for a profile. Uses the
     * posts_profile_status composite index. D-21 query shape.
     */
    export async function loadTwitterUsage(db: Db, profileId: string): Promise<UsageSnapshot> {
      const monthStart = DateTime.utc().startOf('month').toJSDate();

      const [profile] = await db
        .select({
          monthlyBudget: socialProfiles.monthlyTweetBudget,
          warnThresholdPercent: socialProfiles.warnThresholdPercent,
        })
        .from(socialProfiles)
        .where(eq(socialProfiles.id, profileId));

      if (!profile) {
        throw new Error(`Profile ${profileId} not found`);
      }

      const [countRow] = await db
        .select({ c: sql<number>`count(*)::int` })
        .from(posts)
        .where(
          and(
            eq(posts.profileId, profileId),
            gte(posts.publishedAt, monthStart),
            inArray(posts.status, [...COUNTED_STATUSES]),
          ),
        );

      return {
        currentUsage: Number(countRow.c),
        monthlyBudget: profile.monthlyBudget,
        warnThresholdPercent: profile.warnThresholdPercent,
        monthStartUtc: monthStart,
      };
    }

    /**
     * DB-backed pre-flight for single-post publish (LIMIT-01/02/03/04).
     * Loads usage, then delegates the math to the pure shared calculator.
     */
    export async function checkTwitterBudgetWithDb(
      db: Db,
      args: { profileId: string; additionalPostCount: number },
    ): Promise<BudgetCheckResult & { monthStartUtc: Date }> {
      const snapshot = await loadTwitterUsage(db, args.profileId);
      const result = checkTwitterBudget({
        currentUsage: snapshot.currentUsage,
        monthlyBudget: snapshot.monthlyBudget,
        warnThresholdPercent: snapshot.warnThresholdPercent,
        additionalCount: args.additionalPostCount,
      });
      return { ...result, monthStartUtc: snapshot.monthStartUtc };
    }

    /**
     * DB-backed bulk pre-flight (LIMIT-05). Phase 10 CSV upload will call this
     * with additionalCount = parsed row count BEFORE inserting any rows.
     */
    export async function checkBulkBudgetWithDb(
      db: Db,
      args: { profileId: string; additionalCount: number },
    ): Promise<BudgetCheckResult & { monthStartUtc: Date }> {
      const snapshot = await loadTwitterUsage(db, args.profileId);
      const result = checkBulkBudget({
        currentUsage: snapshot.currentUsage,
        monthlyBudget: snapshot.monthlyBudget,
        warnThresholdPercent: snapshot.warnThresholdPercent,
        additionalCount: args.additionalCount,
      });
      return { ...result, monthStartUtc: snapshot.monthStartUtc };
    }
    ```

    **2. `packages/api/src/services/publish-queue.service.ts`** — copy RESEARCH.md Pattern 2 verbatim:

    ```typescript
    import { Queue, type Job } from 'bullmq';
    import type { Redis } from 'ioredis';
    import { QUEUE_NAMES, JOB_NAMES, buildPublishJobId } from '@sms/shared';

    export interface PublishJobPayload {
      postId: string;
      postVersion: number;
      correlationId: string;
    }

    export function createPublishQueueService(redis: Redis) {
      const publishQueue = new Queue<PublishJobPayload>(QUEUE_NAMES.publish, {
        connection: redis,
        defaultJobOptions: {
          removeOnComplete: { count: 100 },
          removeOnFail: { count: 500 },
          attempts: 4,
          backoff: { type: 'publishBackoff' },
        },
      });

      async function enqueuePublish(
        postId: string,
        postVersion: number,
        scheduledAt: Date,
        correlationId: string,
      ): Promise<Job<PublishJobPayload>> {
        const delay = Math.max(0, scheduledAt.getTime() - Date.now());
        return publishQueue.add(
          JOB_NAMES.publishPost,
          { postId, postVersion, correlationId },
          { delay, jobId: buildPublishJobId(postId, postVersion) },
        );
      }

      async function cancelScheduled(postId: string, postVersion: number): Promise<void> {
        const jobId = buildPublishJobId(postId, postVersion);
        const job = await publishQueue.getJob(jobId);
        if (job && (await job.isDelayed())) {
          await job.remove();
        }
      }

      return { publishQueue, enqueuePublish, cancelScheduled };
    }
    ```

    Note the `defaultJobOptions.attempts: 4` (4 total = initial + 3 retries, per D-09 and WORKER-04) and `removeOnFail: { count: 500 }` (keep failed for DLQ inspection, per D-11).
