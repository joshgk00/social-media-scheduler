**Step 1: Fix boundary operators in QueueStatusBadge.tsx**

Change `isInSeasonalPause` to use inclusive boundary comparison matching the backend:

```typescript
function isInSeasonalPause(seasonalStart?: string | null, seasonalEnd?: string | null): boolean {
  if (!seasonalStart || !seasonalEnd) return false;
  const now = new Date();
  const month = String(now.getMonth() + 1).padStart(2, '0');
  const day = String(now.getDate()).padStart(2, '0');
  const today = `${month}-${day}`;

  if (seasonalStart <= seasonalEnd) {
    // Same-year window (e.g., Apr-Oct): paused when outside range (exclusive of boundaries = active)
    return today < seasonalStart || today > seasonalEnd;
  }
  // Cross-year window (e.g., Nov-Jan): paused when in the gap between end and start
  return today > seasonalEnd && today < seasonalStart;
}
```

The research analysis shows the cross-year logic is actually correct for the interior of ranges. The boundary issue is that `<` and `>` (exclusive) mean that on the exact boundary dates (e.g., Nov 1 for seasonalStart='11-01'), the behavior differs from the backend which uses `>=`/`<=`. Fix the operators:

Same-year branch: `today < seasonalStart || today > seasonalEnd` -- on the exact start date, `today < seasonalStart` is false and `today > seasonalEnd` is false, so not paused (correct -- start date is active). On exact end date, same logic. These are already correct because the paused condition is the INVERSE.

Cross-year branch: `today > seasonalEnd && today < seasonalStart` -- on exact end date '01-31', `today > '01-31'` is false, so not paused (correct -- end date is active). On exact start date '11-01', `today < '11-01'` is false, so not paused (correct). On the day AFTER end ('02-01'), `'02-01' > '01-31'` is true AND `'02-01' < '11-01'` is true, so paused (correct). On the day BEFORE start ('10-31'), `'10-31' > '01-31'` is true AND `'10-31' < '11-01'` is true, so paused (correct).

After full trace-through: the exclusive operators produce the CORRECT behavior because `isInSeasonalPause` returns true when OUTSIDE the active window, and boundary dates should be ACTIVE (not paused). So `<`/`>` exclusive is actually correct here.

If all tests pass without code changes, document WR-04 as a false positive from the audit. If any test reveals a bug, fix the specific failing case.
