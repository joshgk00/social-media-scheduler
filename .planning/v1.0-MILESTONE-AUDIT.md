## Executive Summary

1. **Media API routes are dead in production** — `api/src/index.ts` doesn't wire storage backend or transcode queue, so the media router is never mounted

4. **Phase 5 stale dist builds** — 26 test failures from uncompiled TypeScript

## Cross-Phase Integration

### Critical Missing Connections

**INT-01: Media API routes dead in production**

`packages/api/src/index.ts` never instantiates `createStorageBackend()` or creates a transcode BullMQ queue. The conditional guard in `app.ts` (`if (storage && transcodeQueue)`) evaluates to false, and `createMediaRouter` is never mounted. All `/api/media/*` endpoints return 404 at runtime.

- Frontend `useMediaUpload` (XHR to `/api/media/upload`) and `useMediaStatus` (fetch to `/api/media/:id/status`) call dead endpoints
- Worker correctly wires storage and transcode worker, but the API side is disconnected
- **Affected:** MEDIA-01, MEDIA-02, MEDIA-03, MEDIA-04, MEDIA-06, MEDIA-07, MEDIA-08

**INT-02: SESSION_SECRET missing from production Docker Compose**

`packages/api/src/index.ts` calls `requireEnv('SESSION_SECRET')` which throws if the variable is not set. The `docker-compose.yml` api service lists `DATABASE_URL`, `REDIS_URL`, `ENCRYPTION_KEY`, `CSRF_SECRET`, `NODE_ENV` but not `SESSION_SECRET`. Process crashes on startup.

- **Affected:** AUTH-01 through AUTH-07 (all session-dependent flows)

### Phase 5: Queue Engine
- **Stale dist builds** — `@sms/db` and `@sms/shared` packages not rebuilt after Phase 5 additions; 26 test failures across api and worker packages. Fix: `pnpm build` in packages/db and packages/shared

- **WR-04: Cross-year seasonal pause logic inverted** — QueueStatusBadge shows "Active" for Nov-Jan windows during Feb-Oct when it should show "Seasonal pause"
- **WR-05: No error toast on remove from queue** — useRemoveFromQueue has no onError callback; silent failure
- **IN-02: Profile name always shows '-'** — getQueues() doesn't select profile join fields; QueueListItem type declares profile but data is always undefined
- **IN-01: Duplicate ListOrdered import** in Sidebar.tsx (bundler deduplicates silently)
- **IN-03: Logger created per tick** — createLogger('queue-scanner') called inside evaluateQueues on every 60s cycle; should be module-level
